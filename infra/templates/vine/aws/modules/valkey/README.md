# Terraform Module: ElastiCache **Serverless Valkey**

Provision an AWS ElastiCache **Serverless** Valkey cache (Valkey/Redis-compatible) with optional user + user group, Secrets Manager credentials, and a locked-down security group that allows access only from trusted security groups.

This module wraps the official [`terraform-aws-modules/elasticache/aws//modules/serverless-cache`](https://registry.terraform.io/modules/terraform-aws-modules/elasticache/aws/latest/submodules/serverless-cache) and adds:
- Consistent, region-aware naming (`ae1|an1|…` short region codes)
- Optional Valkey **User** and **User Group** (ACLs)
- Optional **AWS Secrets Manager** secret (username/password)
- Security Group that only allows TCP/6379 from provided SGs
- Daily snapshot window and engine/version pins

## Requirements

| Tool | Version |
|---|---|
| Terraform | **>= 1.3** |
| AWS provider | **>= 5.47** |
| random provider | **~> 3.0** |

AWS account permissions sufficient to manage:
- ElastiCache Serverless (cache, users, user groups)
- EC2 security groups
- Secrets Manager (if secrets enabled)

---

## What gets created

- **Serverless Valkey cache** (`engine = "valkey"`) named `${resource_tag}-valkey`
- **Security Group** `valkey-serverless-sg` with:
  - Ingress: TCP/6379 from `var.redis_allowed_security_group_ids`
  - Egress: all
- *(Optional)* **ElastiCache User** and **User Group**:
  - Access string: `on ~* +@all` (full access)
  - Password generated by `random_password` (special chars allowed)
- *(Optional)* **AWS Secrets Manager** secret:
  - Name: `${local.valkey_user_name}`
  - Value: `{ username, password }`

> **Naming:** `resource_tag = <regionShort>-<environment>-<product_name>`, e.g. `ue1-prod-myapp`. Short codes come from `locals.aws_regions_short`.

## Inputs

| Name | Type | Default | Description |
|---|---|---:|---|
| `aws_region` | `string` | n/a | AWS region to deploy to (also used for short code in names). |
| `environment` | `string` | n/a | Environment label (e.g., `dev`, `stg`, `prod`). |
| `product_name` | `string` | n/a | Product / platform name (used in names and tags). |
| `vpc_id` | `string` | n/a | VPC for the cache SG. |
| `subnet_ids` | `list(string)` | `[]` | Subnets used by the Serverless cache. Supply 2+ subnets in distinct AZs. |
| `redis_allowed_security_group_ids` | `list(string)` | n/a | SG IDs that may connect to Valkey on TCP/6379. |
| `snapshot_time` | `string` | `"05:00"` | Daily snapshot time (UTC, HH:MM). |
| `engine_version` | `string` | `"7"` | Valkey major engine version. |
| `data_storage_max` | `number` | `2` | Max data storage (GiB) for Serverless usage limits. |
| `ecpu_per_second_max` | `number` | `1000` | Max eCPU per second for Serverless usage limits. |
| `create_valkey_user_and_secret` | `bool` | `true` | Create ElastiCache User/UserGroup and store creds in Secrets Manager. |

## Quick start

```hcl
terraform {
  required_version = ">= 1.3"
  required_providers {
    aws    = { source = "hashicorp/aws", version = ">= 5.47" }
    random = { source = "hashicorp/random", version = "~> 3.0" }
  }
}

provider "aws" {
  region = var.aws_region
}

module "valkey" {
  # if this code lives in ./modules/valkey, use: source = "./modules/valkey"
  source = "path-to-this-module"

  aws_region  = "us-east-1"
  environment = "prod"
  product_name = "myapp"

  vpc_id     = "vpc-0123456789abcdef0"
  subnet_ids = ["subnet-aaa", "subnet-bbb"]

  redis_allowed_security_group_ids = ["sg-allow-app-ecs"]

  snapshot_time         = "03:00"
  engine_version        = "7"
  data_storage_max      = 4
  ecpu_per_second_max   = 2000
  create_valkey_user_and_secret = true
}
