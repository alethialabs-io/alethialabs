The problem

Going from "we have an app" to "it runs in production on Kubernetes" forces a bad trade:

  • The platform-engineering tax. A real cluster — networking, databases, caching, DNS, secrets, registries, autoscaling, GitOps — is weeks of Terraform/Helm/YAML. Every team rebuilds the same wheel.
  • Rent a black box, or build it yourself. Porter / Qovery / Northflank save you the weeks but make you hand them your cloud keys, run your control plane on their servers, and lock deploys into their abstraction.
  • Credential leakage. Both paths leak long-lived cloud keys and admin kubeconfigs into CI and third-party SaaS — the thing every audit flags.
The solution — the anti-Porter

Alethia gives you the real, standard thing, provisioned for you, that you own. One Spec (your infrastructure design) → a remote runner provisions it in your cloud account, assuming roles at runtime so no credentials are ever stored — and hands you a complete, self-managing, GitOps-wired cluster.

Porter/Qovery/Northflank rent a proprietary PaaS that holds your keys and locks your deploys in. Alethia hands you EKS + ArgoCD + your own tools, provisioned zero-trust. Same outcome — opposite ownership model.

How it works
01Design a Spec. Visual designer or alethia CLI, with a live cost sidebar (Infracost). Generates real OpenTofu you own.
02Connect your cloud zero-trust. The runner runs in your account and assumes scoped roles at execution time. The control plane never sees a key.
03Provision. One deploy stands up EKS + VPC + Aurora + ElastiCache + DynamoDB + SQS/SNS + ECR + S3 + Secrets Manager + Route 53 + WAF.
04Wire GitOps. ArgoCD is installed and connected to your Git repo with auto-sync, plus a full operator suite (external-secrets, external-dns, LB controller, Karpenter, metrics-server).
05Handoff. Kubeconfig, ArgoCD URL, and every endpoint land in your dashboard. git push → your apps deploy and stay reconciled. Alethia is never in the data path.
Four pillars
Zero-trust provisioning. runner assumes roles at runtime; control plane stores nothing.
Own it / self-hostable. Own the cluster; self-host Alethia as ~4 containers.
GitOps app-delivery. Standard ArgoCD + your repo — not a proprietary pipeline.
Pluggable + multi-cloud. Swap any integration category, span many clouds.
What you get — one deploy
  • Cluster + network: managed Kubernetes (EKS) on a VPC with public/private subnets, security groups, IAM/IRSA, optional Karpenter autoscaling.
  • Data + ops services: Aurora PostgreSQL, ElastiCache Redis, DynamoDB, SQS/SNS, ECR, S3, Secrets Manager (KMS), Route 53 + ACM, optional WAF.
  • GitOps, wired — not just installed: ArgoCD connected to your repo with auto-sync (prune + self-heal). Push → deploy → stay reconciled.
  • A real operator suite via app-of-apps: external-secrets, external-dns, AWS load-balancer-controller, Karpenter, metrics-server, default storage class.
  • The handoff: cluster endpoint + kubeconfig, ArgoCD URL + admin access, and every database/cache endpoint — all in your account.

Business model — open-core

AGPL core (provisioning + GitOps + integrations + single-tenant + community RBAC) is free and self-hostable. Commercial ee/ adds orgs/RBAC-at-scale, SSO (SAML/OIDC), audit, multi-tenancy. Modeled on Cal.com.
