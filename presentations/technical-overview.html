Alethia web + Postgres

Designs Specs, queues jobs, stores config and identifiers — never cloud secrets.

Execution plane

02 · Self-Host Architecture — The Small Package

Four containers on commodity infra.

No dependency on any single SaaS. The operator supplies one Postgres URL, one S3 endpoint + keys, and OAuth secrets — nothing else.

app

Next.js

Better Auth (MIT) · Drizzle (Apache-2.0) · SSE. Routes for auth, CLI, jobs, stream.

postgres

Postgres

App + auth + queue RPCs + LISTEN/NOTIFY. Portable; the PostgreSQL license.

s3

SeaweedFS

Any S3-compatible store (Apache-2.0, swappable). TF state + plan artifacts.

node

Go runner

Already S3-native (path-style, lockfile — no DynamoDB). Pulls jobs over HTTP.

Optional  redis is a scale-out profile only — the default profile never starts it.

Replaces Supabase's ~10 containers (GoTrue, PostgREST, Realtime, Storage, Kong, Studio…) across four independent subsystems.

02b · Realtime Without Redis

SSE backed by Postgres LISTEN/NOTIFY.

Live job-log streaming and status, multi-instance, with no message bus.

runner POST ─► insert_job_log() ─► job_logs
                       │
              pg_notify('job_logs',{jobId,logId})
                       │  (IDs only, 8KB cap)
                       ▼
  each app instance: ONE LISTEN conn ──► SSE fan-out
                       ▼
  Browser: EventSource('/api/stream/jobs/:id')
           on reconnect → fetch logs since last logId

Multi-instance, no Redis

One LISTEN connection per app instance fans out to its SSE clients — horizontal scale with no shared bus.

No message-loss regression

The existing 3-second status poll stays as the reconciliation backstop. Heartbeat comment every ~20 s.

Scale-out seam

A Redis pub/sub impl sits behind a RealtimeTransport interface for very-high-fan-out / hosted tiers.

04 · Authorization — One PDP, Two Backends

One can(). Two engines. An RLS blast wall.

lib/authz/ is the only place an access decision is made — so swapping the engine never touches the ~200 call sites.

Community

PostgresRbacPDP

Scoped RBAC as plain Drizzle queries. Walks Org→Zone→Spec inheritance in SQL (recursive CTE). Zero extra services.

Enterprise

OpenFgaPDP

OpenFGA (Apache-2.0, CNCF Incubating), Postgres-native, embeddable as a Go lib. ListObjects for list views; ABAC conditions on sensitive actions.

Backstop

Coarse org_id RLS

One policy per table: org_id = current_setting('app.current_org'), set transaction-scoped. Unbypassable — even raw SQL can't cross tenants.

interface PDP {
  can(actor, action, resource)             // Decision
  enforce(actor, action, resource)         // throws 403 — default for routes
  bulkCheck(actor, checks[])               // batched — never loop can()
  listAccessible(actor, action, type)      // the ListObjects equivalent — mandatory per list view
}

Engine swap = a PDP binding flip + a tuple backfill, no call-site rewrites. CI lint forbids authz logic (auth.uid(), .eq('user_id')) outside lib/authz/.

07 · Security Posture — Summary

Zero-trust by construction; defense in depth.

No stored cloud secrets

Control plane stores identifiers only. The runner assumes roles at execution time — AssumeRole / WIF / federated identity per cloud.

Two-layer authorization

Rich PDP for hierarchical / ABAC decisions; coarse org_id RLS as the unbypassable tenant blast wall. Default-deny everywhere.

Bounded runner authority

A runner acts as the user who queued the job; service-role DB connection bypasses RLS but is gated by verifyRunnerToken + a PDP check bound to that user.

Runtime-only credentials

Integration secrets are fetched at execution and injected as TF vars — never in the config_snapshot, never at rest in the queue.

Audit at the decision point

authz_audit_log is written by the PDP on every enforce() — audit can't be forgotten at call sites.

Permissive supply chain

Better Auth (MIT), Drizzle / SeaweedFS (Apache-2.0), OpenTofu (MPL-2.0). License-scan CI keeps the graph clean. AGPL is our code only.

Exit criterion: adversarial cross-tenant test denied at both the PDP and the database.
