What Alethia is

The real thing, provisioned for you — that you own.

Getting from "we have an app" to "it runs in production on Kubernetes" forces a bad trade: rent a black box that holds your keys, or rebuild weeks of Terraform yourself. Alethia removes the trade.

The trade today

Rent a black box, or build it yourself

Porter / Qovery / Northflank save you the weeks — but make you hand over cloud keys, run your control plane on their servers, and lock deploys into their abstraction. The DIY route keeps ownership but brings back the platform-engineering tax.

The Alethia model

One Spec → a cluster you own

Your Spec (infra design) → a remote runner (worker) provisions it in your cloud, assuming roles at runtime so no credentials are ever stored. It hands you a self-managing, GitOps-wired cluster. Alethia never sits in the data path.

Same outcome (production platform, fast). Opposite ownership model — the anti-Porter.

The four pillars

What makes Alethia different

01

Zero-trust provisioning

The runner assumes cloud roles at execution time. The control plane never sees or stores your keys — only identifiers.

02

Own it / self-hostable

You own the cluster — and can self-host the Alethia control plane as ~4 containers. No Supabase, no SaaS dependency.

03

GitOps app-delivery

A real cluster + ArgoCD wired to your repo, standard Kustomize/Helm — not a proprietary pipeline you can't leave.

04

Pluggable + multi-cloud

Bring your own tools per category — DNS, secrets, observability, registry — across many clouds and cluster strategies.

What you get · verified, end-to-end

One deploy wires the whole platform

Cluster + network

Managed Kubernetes (EKS today) on a VPC with public/private subnets, security groups, IAM/IRSA, optional Karpenter autoscaling.

Data + ops services

Aurora PostgreSQL, ElastiCache Redis, DynamoDB, SQS/SNS, ECR, S3, Secrets Manager (KMS), Route 53 + ACM, optional WAF.

GitOps — wired

ArgoCD installed and connected to your Git repo, auto-sync with prune + self-heal. Push → apps deploy and stay reconciled.

Operator suite

Via app-of-apps: external-secrets, external-dns, AWS load-balancer-controller, Karpenter, metrics-server, a default storage class.

The handoff

Cluster endpoint + kubeconfig · ArgoCD URL + admin access · every database/cache endpoint — all in your dashboard. It's your cluster in your account. Alethia doesn't sit in the data path.

Pluggable integrations

Bring the tools you already use

Decouple what a Spec needs from who provides it. The cloud-native option is just the default per category — swap in your own, per Spec.

CategoryDefault (cloud-native)Pluggable alternatives
DNSRoute 53 / Cloud DNS / Azure DNSCloudflare
SecretsSecrets Manager / Key VaultHashiCorp Vault
ObservabilityCloudWatch / Cloud MonitoringDatadog · Grafana Cloud · Prometheus
RegistryECR / Artifact Registry / ACRDocker Hub

A data-driven catalog is the registry of record. Adding a provider is a catalog row + a CategoryProvider impl + a module + a credential mapping — no core changes. Credentials flow to the runner at runtime only, never into the config snapshot.

redis is an optional scale-out profile only. The operator provides one Postgres URL, one S3 endpoint + keys, and OAuth secrets — nothing else.

De-Supabase, permissive by design

Auth → Better Auth (MIT) · Data → Drizzle (Apache-2.0) · Realtime → SSE over Postgres LISTEN/NOTIFY · Storage → SeaweedFS (Apache-2.0). No single-SaaS dependency; AGPL is our code only.

git clone → cp .env.example .env → docker compose up. You run the entire control plane.

Release trajectory

V1 · Launch

Provision & Own

Deliver the complete, GitOps-wired cluster you own. The app experience is ArgoCD — standard, yours. A thin dashboard adds visibility — sync status, cost, health — over what you own, plus the integration-catalog breadth.

The sharp, shippable wedge: be the anti-Porter.

V2 · Later

Provision & Operate

An Alethia-native day-2 experience — deploys, logs, rollbacks, preview environments, ongoing cluster management — rivaling Porter's polish, while staying self-hostable and zero-trust.

Same ownership promise, fuller DX.

Open source & business model

AGPL-3.0, open-core. The self-hostable core — provisioning + GitOps + integrations + single-tenant + community RBAC — is free and complete. Orgs / SSO / RBAC-at-scale / audit / multi-tenancy + hosted SaaS are the commercial ee/ tier.
