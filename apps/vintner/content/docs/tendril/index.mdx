# Tendril Agent

Tendril is the execution engine of the platform. It is a Go binary packaged as a Docker container that runs in AWS ECS Fargate (ARM64). Tendrils claim jobs from the Trellis queue, assume temporary cloud credentials, execute Terraform, install ArgoCD, and stream logs back in real-time.

## Key Facts

| Property | Value |
|----------|-------|
| Language | Go |
| Runtime | ECS Fargate (ARM64 / Graviton) |
| Container registries | AWS ECR + GitHub Container Registry |
| Versioning | release-please (`tendril-v*` tags) |
| Embedded tools | Terraform, kubectl, Helm, AWS CLI, gcloud, Azure CLI, Infracost |
| Poll interval | 10 seconds |
| Heartbeat interval | 30 seconds |
| Idle scale-down | After 5 minutes (Lambda scaler) |

## Two Operating Modes

| Mode | Description |
|------|-------------|
| **Cloud-hosted** | Deployed as ECS Fargate task, managed by the platform. Automatic scaling, no user setup required. |
| **Self-hosted** | User runs the `tendril` binary on their own infrastructure. For on-premises or air-gapped environments. |

## What Tendril Does

1. **Polls for jobs** — every 10 seconds, calls `POST /api/jobs/claim`
2. **Claims atomically** — PostgreSQL `claim_next_job()` with `FOR UPDATE SKIP LOCKED` prevents double-execution
3. **Assumes credentials** — STS AssumeRole (AWS), WIF token exchange (GCP), or OIDC federated identity (Azure)
4. **Executes Terraform** — `init` → `plan` → `apply` or `destroy`
5. **Installs ArgoCD** — Helm chart deployment after successful apply
6. **Streams logs** — batched POST every 2 seconds → Supabase Realtime → browser
7. **Reports results** — updates job status with execution metadata, timing, and outputs
8. **Sends heartbeat** — every 30 seconds to confirm it is alive

## Container Contents

The Tendril Docker image includes:

- **Go binary** — the tendril executable
- **Terraform** — for infrastructure provisioning
- **kubectl** — for Kubernetes cluster access
- **Helm** — for ArgoCD and component installation
- **Infrastructure templates** — all AWS, GCP, and Azure Terraform modules (embedded via `go embed` or staged during Docker build)
- **ArgoCD templates** — Helm values and Application manifests
