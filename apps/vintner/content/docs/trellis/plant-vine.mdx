# Plant a Vine

The Plant a Vine form is the primary way to define infrastructure. It has 11 sections covering everything from networking to GitOps repositories. The form adapts dynamically based on the selected cloud provider.

## Provider Ribbon

At the top of the form, a **Provider Ribbon** lets you switch between AWS, GCP, and Azure. When you switch providers:

- Form fields adapt (EKS ↔ GKE ↔ AKS, Aurora ↔ Cloud SQL ↔ Azure Database, etc.)
- Instance types change to provider-specific options
- Region lists update to the provider's available regions
- Cost estimates recalculate using the provider's pricing

## Cloud Identity Selector

Before configuring infrastructure, you must select a **Cloud Identity** — a connected cloud account. If no identities are connected, the selector is empty and links to the Integrations page.

## Form Sections

### 1. Project Basics

| Field | Description |
|-------|-------------|
| Project Name | Infrastructure project identifier (used in resource naming) |
| Provider | AWS / GCP / Azure (via Provider Ribbon) |
| Region | Cloud region (e.g., eu-west-1, us-central1, westeurope) |
| Environment Stage | development / staging / production |
| Terraform Version | Default: 1.11.4 |

| Field | Description |
|-------|-------------|

| Infra template repo | Source template for Terraform modules |
| Infra destination repo | Where generated Terraform is pushed |

| GitOps destination repo | Where rendered manifests are pushed |
| App template repo | Source template for application boilerplate |
| App destination repo | Where rendered application code is pushed |

## Cost Sidebar

As you fill in the form, a **Cost Sidebar** on the right shows the estimated monthly cost. It updates in real-time as fields change. See [Cost Estimation](/docs/trellis/cost-estimation) for details.

## What Happens on Submit

When you click "Plant Vine":

1. A `vines` row is created with status `DRAFT`
2. Component rows are created in the relevant tables (`vine_network`, `vine_cluster`, `vine_databases`, etc.)
3. The vine appears in the vineyard's vine listing
4. The user can now run Plan → Apply from the vine detail page
