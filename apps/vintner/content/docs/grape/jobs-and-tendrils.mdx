# Jobs & Tendrils

Grape provides full job monitoring and Tendril management from the terminal.

## Monitoring Jobs

### List All Jobs

```bash
grape jobs list
```

Output:

```
ID            TYPE     STATUS      VINE            CREATED
a1b2...       PLAN     SUCCESS     api-backend     2 min ago
c3d4...       DEPLOY   PROCESSING  api-backend     30 sec ago
e5f6...       DESTROY  QUEUED      data-platform   just now
```

### Get Job Details

```bash
grape jobs get --id <job-id>
```

Shows full job information: type, status, assigned Tendril, timestamps, error message (if failed), and execution metadata.

### Stream Logs

```bash
grape jobs logs --id <job-id> --follow
```

The `--follow` flag continuously streams new log entries as the Tendril writes them — similar to `tail -f`. Without `--follow`, it shows all logs written so far and exits.

Log output includes:
- Terraform plan/apply output
- Resource creation progress
- Error messages and warnings
- System messages (credential assumption, phase transitions)

### Wait for Completion

```bash
grape jobs wait --id <job-id>
```

Blocks until the job reaches a terminal state (SUCCESS, FAILED, or CANCELLED). Exits with code 0 on success, non-zero on failure. Useful in scripts:

```bash
grape plan --vine-id <id>
# capture job ID from output
grape jobs wait --id <plan-job-id> && grape harvest --vine-id <id> --plan-job-id <plan-job-id>
```

### Cancel a Job

```bash
grape jobs cancel --id <job-id>
```

Cancels a QUEUED or PROCESSING job. If the job is currently being executed by a Tendril, the Tendril detects the cancellation and stops Terraform gracefully.

## Managing Tendrils

### List Tendrils

```bash
grape worker list
```

Output:

```
ID          NAME                STATUS   MODE           VERSION   LAST HEARTBEAT
a1b2...     eu-west-1-tendril   ONLINE   cloud-hosted   0.1.0     5 sec ago
c3d4...     self-hosted-dev     OFFLINE  self-hosted    0.1.0     2 min ago
```

### View Configuration

```bash
grape worker config
```

Shows the current Tendril configuration including API endpoint, mode, and connection status.

### Remove a Tendril

```bash
grape worker remove --id <worker-id>
```

For cloud-hosted Tendrils, this queues a `DESTROY_WORKER` job that tears down the ECS infrastructure. For self-hosted Tendrils, it simply removes the registration from Trellis.

## Listing Clusters

```bash
grape clusters list
```

Shows all provisioned Kubernetes clusters with their name, provider, region, endpoint, and ArgoCD URL.
