# Grape CLI

Grape is the command-line interface for the Trellis platform. It provides terminal-based access to all infrastructure operations: creating vineyards, planning and deploying vines, monitoring jobs, and managing Tendrils.

## Overview

- **Language:** Go
- **Framework:** Cobra (commands) + Charmbracelet (TUI: lipgloss, huh, bubbletea)
- **Current version:** 0.1.7 (managed by release-please)
- **Distribution:** Homebrew tap + GitHub Releases (linux/darwin, amd64/arm64)

## Command Categories

| Category | Commands | Purpose |
|----------|----------|---------|
| **Auth** | `login`, `logout` | Device code authentication |
| **Vineyards** | `vineyard create`, `vineyard list`, `vineyard delete` | Workspace management |
| **Vines** | `vine list`, `vine get` | Infrastructure configuration inspection |
| **Operations** | `harvest`, `plan`, `destroy` | Queue provisioning jobs |
| **Jobs** | `jobs list`, `jobs get`, `jobs logs`, `jobs cancel`, `jobs wait` | Monitor execution |
| **Clusters** | `clusters list` | View provisioned clusters |
| **Tendrils** | `worker list`, `worker config`, `worker remove` | Agent management |
| **Selectors** | `selectors` | Interactive vineyard/vine picker |

## How It Works

Grape communicates with the Trellis API over HTTPS. It does not run Terraform directly — instead, it queues jobs that Tendrils execute. The CLI is a thin client for:

1. **Authentication** — Device code flow, token stored locally
2. **Configuration** — Creating and inspecting vineyards/vines
3. **Job submission** — Queuing Plan/Deploy/Destroy jobs
4. **Monitoring** — Streaming logs, checking job status
5. **Management** — Listing clusters, managing Tendrils

## Quick Start

```bash
# Install via Homebrew
brew install bobikenobi12/bb-thesis-2026/grape

# Authenticate
grape login

# List vineyards
grape vineyard list

# Queue a plan for a vine
grape plan --vine-id <id>

# Watch logs
grape jobs logs --id <job-id> --follow

# Deploy (harvest)
grape harvest --vine-id <id> --plan-job-id <plan-job-id>
```
