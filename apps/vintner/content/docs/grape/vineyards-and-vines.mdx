# Vineyards & Vines

Grape provides commands for managing vineyards (workspaces) and inspecting vines (infrastructure configurations). The primary provisioning workflow follows a Plan-Review-Apply pattern.

## Creating a Vineyard

```bash
grape vineyard create --name api-backend
```

This creates the vineyard in Trellis and returns its UUID. Vineyards are the top-level grouping — all vines belong to a vineyard.

## Listing Vineyards

```bash
grape vineyard list
```

Output:

```
ID                                   NAME          VINES  CREATED
a1b2c3d4-...                         api-backend   2      2026-06-01
e5f6g7h8-...                         data-platform 1      2026-05-28
```

## Inspecting Vines

List all vines in a vineyard:

```bash
grape vine list --vineyard-id a1b2c3d4-...
```

Get detailed vine info:

```bash
grape vine get --id <vine-id>
```

This shows the vine's status, region, environment, cloud provider, and component summary.

## Plan-Review-Apply Workflow

The standard infrastructure provisioning workflow from the CLI:

### Step 1: Plan

```bash
grape plan --vine-id <vine-id>
# Job created: <plan-job-id> (QUEUED)
```

This queues a PLAN job. A Tendril claims it and runs `terraform plan`. Wait for completion or stream logs:

```bash
grape jobs logs --id <plan-job-id> --follow
```

### Step 2: Review

Once the plan completes, review what will be created:

```bash
grape jobs get --id <plan-job-id>
```

The output shows:
- Number of resources to add, change, destroy
- Estimated monthly cost (if Infracost is configured)
- Plan artifact reference

### Step 3: Apply (Harvest)

If the plan looks good, apply it:

```bash
grape harvest --vine-id <vine-id> --plan-job-id <plan-job-id>
# Job created: <deploy-job-id> (QUEUED)
```

The `--plan-job-id` flag ensures the apply uses the exact plan you reviewed. If the vine configuration has changed since the plan, the job will fail with a hash mismatch error.

Stream deployment logs:

```bash
grape jobs logs --id <deploy-job-id> --follow
```

### One-Shot with `--wait`

For CI/CD or scripting, combine with `--wait` to block until completion:

```bash
grape plan --vine-id <id> --wait
grape harvest --vine-id <id> --plan-job-id <id> --wait
```

## Destroy Workflow

Tear down infrastructure:

```bash
grape destroy --vine-id <vine-id>
```

This queues a DESTROY job. The Tendril runs `terraform destroy` and cleans up all provisioned resources.

## Interactive Selection

When `--vine-id` is omitted from `plan`, `harvest`, or `destroy`, Grape launches an interactive TUI:

1. First, select a vineyard from the list
2. Then, select a vine within that vineyard
3. The command proceeds with the selected vine

This is powered by the Charmbracelet `huh` form library and provides a smooth terminal experience. See [Interactive Selectors](/docs/grape/selectors) for details.
