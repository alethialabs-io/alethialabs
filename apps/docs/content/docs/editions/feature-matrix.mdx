# Feature matrix

The single source of truth for what each edition includes. <Community>Community</Community> is free and
open-source (AGPL-3.0); <Enterprise>Enterprise</Enterprise> is licensed.

## Authorization core

Everything here is **open-source** — the authorization model is the same in both editions.

| Capability | Community | Enterprise |
|---|:--:|:--:|
| Policy Decision Point (`can` / `enforce` / `bulkCheck` / `listAccessible`) | ✅ | ✅ |
| Built-in roles — owner · admin · operator · viewer | ✅ | ✅ |
| Per-resource grants + Org → Zone → Spec inheritance | ✅ | ✅ |
| Explicit **deny** with hierarchy + exceptions | ✅ | ✅ |
| Single-permission grants (not just roles) | ✅ | ✅ |
| Row-Level Security tenant isolation (the blast wall) | ✅ | ✅ |
| Authorization **audit log** (denials + sensitive actions) | ✅ | ✅ |
| Registry-as-code (resources × actions, exhaustively typed) | ✅ | ✅ |

<Callout type="info">
  The OpenFGA engine is **optional even with a license** — Enterprise runs perfectly on `PostgresRbacPDP`.
  Turn OpenFGA on when you want a dedicated relationship-graph engine. See the
  [OpenFGA operator guide](/access-control/openfga).
</Callout>

## How the line is drawn

Community is a **complete single-tenant product**: roles, grants, hierarchy, deny, the PDP, RLS, and the audit
log all ship open-source. Enterprise is about **organizations of people** — inviting and grouping members,
authenticating them with SSO, authoring fine-grained custom roles, assigning access through the portal, and
(optionally) deciding it with OpenFGA.
