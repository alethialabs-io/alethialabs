# Terraform modules

The bundle runs on **any VM**, so the simplest path is: make a VM on your cloud,
point DNS at it, and run the [installer](./index). The Terraform modules below
automate that "VM + firewall + DNS + Docker" step per cloud — each one provisions
a host and runs the same bundle via cloud-init.

## Available now

| Cloud | Module | Notes |
|---|---|---|
| **Hetzner** | `infra/hetzner` | CAX21 ARM (~€10–11/mo) — cheapest. `hcloud` + Cloudflare DNS. |
| **AWS** | `infra/aws-cp` | EC2 `t4g.medium` Graviton (~$36/mo) + EIP. |

Usage (Hetzner shown):

```bash
cd infra/hetzner
cp backend.hcl.example backend.hcl        # S3-compatible state backend
cp terraform.tfvars.example terraform.tfvars   # tokens, domain, ssh key
terraform init -backend-config=backend.hcl
terraform apply
# → set DNS / DEPLOY_HOST to the output IP, then run the installer or the deploy workflow
```

Both modules take a Cloudflare API token + zone for DNS and an SSH public key. The
private key is how CI (or you) deploy updates over SSH.

| Cloud | Status |
|---|---|

All follow the same thin shape (VM + cloud-init + DNS) and run the identical
compose bundle — no per-cloud divergence. A Helm chart (any managed Kubernetes)
and one-click "Deploy to…" buttons are also on the roadmap.
