# Alibaba (`infra/cp-alibaba`)

A single x86 **`ecs.g7.large`** (2 vCPU / 8 GB) ECS instance fronted by a Cloudflare Tunnel.
**≈ $45–65/mo** (approx, region-dependent).

## What it creates
- `alicloud_instance` (`ecs.g7.large`, Ubuntu 24.04 x86_64) with a 40 GB **encrypted** `cloud_essd`
  system disk
- `alicloud_vpc` + `alicloud_vswitch` (AZ auto-picked to one that has the x86 type)
- `alicloud_security_group` with **no ingress rules** — web is the tunnel, admin is Session Manager
- `cloudflare_zero_trust_tunnel_cloudflared` (+ config) and **proxied** CNAMEs (apex + `www`)
- cloud-init via `user_data` (installs Docker, clones the repo)

## Access — Session Manager / Cloud Assistant (no open port)
The Cloud Assistant agent (preinstalled on the Ubuntu image) dials out:
- **Interactive:** ECS Session Manager (Alibaba console → ECS → the instance → *Connect*).
- **Automation:** `aliyun ecs RunCommand`.

## Prerequisites
- An Alibaba Cloud account with a **RAM access key** (ECS + VPC permissions).
- A Cloudflare **API token** (DNS edit + Zero Trust), the **zone id**, and the **account id**.

## Apply
```bash
cd infra/cp-alibaba
cp backend.hcl.example backend.hcl
cp terraform.tfvars.example terraform.tfvars  # region, cloudflare_* (token/zone/account)
export ALICLOUD_ACCESS_KEY=... ALICLOUD_SECRET_KEY=...
tofu init -backend-config=backend.hcl
tofu plan && tofu apply
export TUNNEL_TOKEN="$(tofu output -raw tunnel_token)"
```

## CI secrets (`infra-cp-alibaba.yml`)
`ALICLOUD_ACCESS_KEY`, `ALICLOUD_SECRET_KEY` (RAM key), `CLOUDFLARE_API_TOKEN`, `CLOUDFLARE_ZONE_ID`,
`CLOUDFLARE_ACCOUNT_ID`, and the `TF_STATE_S3_*` backend secrets. `apply` is gated behind
`vars.INFRA_ALIBABA_APPLY`.

## Tuning
`instance_type` (`ecs.g7.large` default — **must be x86**), `system_disk_size`, `region`, `zone_id`
(leave empty to auto-pick). An auto-assigned public IP handles egress.
