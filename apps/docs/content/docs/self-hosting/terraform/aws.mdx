# AWS (`infra/cp-aws`)

A single x86 **EC2 `t3.large`** (2 vCPU / 8 GB) fronted by a Cloudflare Tunnel — **no bastion, no
open port**. **≈ $72/mo** (t3.large + 60 GB gp3; Infracost, on-demand, `eu-west-1`; `t3.medium` ≈ half if 4 GB fits).

## What it creates
- `aws_instance` (`t3.large`, Ubuntu 24.04 amd64) with an encrypted 60 GB gp3 root; **IMDSv2 required**
- `aws_security_group` with **no ingress** (egress only) — web is the tunnel, SSH is SSM
- IAM role + instance profile with **`AmazonSSMManagedInstanceCore`** only (Session Manager)
- `cloudflare_zero_trust_tunnel_cloudflared` (+ config) and **proxied** CNAMEs (apex + `www`)
- cloud-init: installs Docker, clones the repo (Docker data on the durable EBS root)

## Access — SSM Session Manager
No SSH port is open. The SSM agent (preinstalled on the AMI) dials out; connect with:
```bash
aws ssm start-session --target <instance_id>
```

## Prerequisites
- AWS credentials for the target account (`AWS_ACCESS_KEY_ID` / `AWS_SECRET_ACCESS_KEY`).
- A Cloudflare **API token** with **DNS edit + Zero Trust (Tunnel)** perms, the **zone id**, and the
  **account id**.

## Apply
```bash
cd infra/cp-aws
cp backend.hcl.example backend.hcl
cp terraform.tfvars.example terraform.tfvars # cloudflare_* (token/zone/account), domain, aws_region
export AWS_ACCESS_KEY_ID=… AWS_SECRET_ACCESS_KEY=…
tofu init -backend-config=backend.hcl
tofu plan && tofu apply
export TUNNEL_TOKEN="$(tofu output -raw tunnel_token)"
```

## CI secrets (`infra-cp-aws.yml`)
`AWS_ACCESS_KEY_ID`, `AWS_SECRET_ACCESS_KEY` (provider), `CLOUDFLARE_API_TOKEN`, `CLOUDFLARE_ZONE_ID`,
`CLOUDFLARE_ACCOUNT_ID`, and the `TF_STATE_S3_*` backend secrets. `apply` is gated behind
`vars.INFRA_AWS_APPLY`.

## Tuning
`instance_type` (`t3.large` default — **must be x86**), `root_volume_size`, `aws_region`. Runs in the
account's default VPC with an auto-assigned public IP (egress only).
