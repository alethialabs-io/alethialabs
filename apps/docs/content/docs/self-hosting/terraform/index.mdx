# Terraform modules

The repeatable IaC path. Each cloud has a self-contained OpenTofu stack under `infra/cp-<cloud>/`
that provisions **one** control-plane VM and fronts it with a **Cloudflare Tunnel**, so the box has
**no public web ports**. It lands on the same [bundle](../index) as the [one-click](../one-click) VM
and the [Helm](../helm) chart — pick this when you want version-controlled infrastructure with a
plan/apply CI loop.

> **Pick one.** The stacks all serve the **same domain/zone**, so they are alternatives — apply
> **one instead of another**, never two at once (two would fight over the apex DNS + tunnel). The
> hosted alethialabs.io runs on Hetzner.

## The model — closed origin, zero standing inbound

Every stack follows the same shape:

- **Ingress is a Cloudflare Tunnel.** `cloudflared` (a service in the bundle) dials **out**; nothing
  dials in. The firewall / security group exposes **no 80/443**. Apex and `www` are **proxied**
  `CNAME`s onto `<tunnel_id>.cfargotunnel.com`, and TLS terminates at Cloudflare's edge.
- **The connector token is an output.** The stack exports `tunnel_token` (sensitive); set it as
  `TUNNEL_TOKEN` in the box's `.env` and the compose `cloudflared` connects with nothing else
  (`config_src = "cloudflare"`, remotely-managed).
- **The public IP is egress-only** (image pulls, `git`, dialing the tunnel out); inbound is closed.
- **Hardened + asserted.** Each stack ships a `checks.tf` that fails the plan if the posture regresses
  (no web ingress, disk encryption / verified boot, tunnel remotely-managed).

## Shared workflow

Every module is the same three steps:

```bash
cd infra/cp-<cloud>
cp backend.hcl.example backend.hcl          # S3-compatible state backend (endpoint + creds)
cp terraform.tfvars.example terraform.tfvars # cloud creds, domain, cloudflare_* ids
tofu init -backend-config=backend.hcl
tofu plan && tofu apply
```

Then wire the tunnel token into the bundle `.env`:

```bash
export TUNNEL_TOKEN="$(tofu output -raw tunnel_token)"
```

State lives in an S3-compatible bucket (`TF_STATE_S3_*`), one key per stack
(`<cloud>-cp/terraform.tfstate`). DNS records are **proxied** CNAMEs onto the tunnel — there is no
open origin. See each cloud's page for the exact GitHub secrets.

## Prerequisites

- A **Cloudflare account + zone** and an API token with **DNS edit** and **Zero Trust (Cloudflare
  Tunnel)** permissions — you pass `cloudflare_account_id` and `cloudflare_zone_id`.
- An **S3-compatible state backend** (`backend.hcl`).
- The chosen cloud's own credentials.

## CI

Each stack has a workflow (`infra-cp-hetzner.yml`, `infra-cp-aws.yml`, `infra-cp-gcp.yml`,
`infra-cp-azure.yml`, `infra-cp-alibaba.yml`): the PR **`plan`** job runs `tofu validate` + fmt/tflint/
Trivy; the **`apply`** job is **gated off** behind `vars.INFRA_<CLOUD>_APPLY` (prod runs on Hetzner).

## Not managed here

Cloudflare **Email Routing** is zone-global and lives only in `cp-hetzner` (see [Email](../email)),
so the stacks don't collide on the shared zone.
