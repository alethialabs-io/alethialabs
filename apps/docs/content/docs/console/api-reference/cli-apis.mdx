# CLI APIs

These endpoints are called by the alethia CLI. Authentication is via `Authorization: Bearer {jwt}` header with a CLI-issued JWT. See [Authentication Flows](/concepts/authentication).

## Configurations (Projects)

### `GET /api/cli/configurations`
List all project configurations for the authenticated user.

### `GET /api/cli/configurations/{name}`
Get a configuration by name.

### `GET /api/cli/configurations/by-project-name/{project_name}`
Look up a project by name.

### `POST /api/cli/configurations`
Create a new project configuration from CLI input.

## Jobs

### `POST /api/jobs`
Create a new job. Accepts:

```json
{
  "job_type": "PLAN",
  "configuration_id": "project-uuid",
  "assigned_runner_id": "runner-uuid",
  "plan_job_id": "plan-job-uuid"
}
```

Valid job types: PLAN, DEPLOY, DESTROY, DESTROY_RUNNER.

If `configuration_id` is provided, the server auto-resolves `cloud_identity_id`, computes config hash, and creates a config snapshot.

### `GET /api/jobs`
List all jobs for the authenticated user. Supports a `?status={status}` filter.

### `GET /api/cli/jobs/{id}`
Get job status and metadata.

### `GET /api/cli/jobs/{id}/logs`
Get job logs (same format as the Runner API).

### `POST /api/cli/jobs/{id}/cancel`
Cancel a QUEUED or PROCESSING job.

## Runners

### `GET /api/cli/runners`
List all registered Runners.

### `POST /api/cli/runners/register`
Register a self-operated Runner. Always creates `operator=self`, `provisioning=registered` (no `mode` body field).

### `POST /api/cli/runners/deploy`
Deploy a new self-operated Runner into the customer's cloud (`operator=self`, `provisioning=deployed`).

### `DELETE /api/cli/runners/{id}`
Remove a Runner's database record.

## Repositories

### `GET /api/cli/repositories/github`
List GitHub repositories for the authenticated user.

### `GET /api/cli/repositories/gitlab`
List GitLab repositories.

### `GET /api/cli/repositories/bitbucket`
List Bitbucket repositories.

## Releases

### `POST /api/releases/runner`
Publish a new Runner release (CI/CD only, requires `RELEASE_API_SECRET`).

**Body:**
```json
{
  "version": "0.1.0",
  "release_notes": "Changelog content..."
}
```

### `POST /api/runners/register`
Register a Runner (CI/CD only, requires `RELEASE_API_SECRET`). Used by Terraform during Runner deployment; always creates an `operator=managed` runner (no `mode` body field).
