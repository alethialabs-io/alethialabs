# Clusters

The Clusters page (`/dashboard/clusters`) shows all provisioned Kubernetes clusters from active projects.

## Cluster Cards

Each cluster displays:

| Field | Description |
|-------|-------------|
| **Project Name** | The project's name + a **derived** status badge (Active / Reconciling / Degraded / Provisioning) — rolled up from the cluster and every data service, never hardcoded to "Active" |
| **Provider** | Provider icon with region, environment, and Kubernetes version (e.g., `K8s 1.32`) |
| **Cluster Endpoint** | API server URL (copy button) |
| **kubeconfig Command** | Configure kubectl (copy button) — shown for **AWS** clusters; GCP/Azure users run their own CLI (see below) |
| **ArgoCD URL** | External link to the ArgoCD dashboard, plus a command to retrieve the admin password |
| **Components** | Control plane, databases, caches, and DNS — each with its endpoint (copy buttons) and its own derived status |

### kubeconfig Commands

The Console generates the ready-to-copy command for **AWS** clusters. For GCP and Azure, use the equivalent CLI command below:

<Tabs items={["AWS", "GCP", "Azure"]}>
  <Tab value="AWS">
    ```bash
    aws eks update-kubeconfig --name {cluster-name} --region {region}
    ```
  </Tab>
  <Tab value="GCP">
    ```bash
    gcloud container clusters get-credentials {cluster-name} --region {region}
    ```
  </Tab>
  <Tab value="Azure">
    ```bash
    az aks get-credentials --name {cluster-name} --resource-group {resource-group}
    ```
  </Tab>
</Tabs>

## Pre-Installed Components

Every cluster comes with these components installed via ArgoCD (see [GitOps & ArgoCD](/concepts/gitops-argocd)):

| Component | Purpose |
|-----------|---------|
| **ArgoCD** | GitOps continuous delivery |
| **AWS Load Balancer Controller** | Ingress via ALB/NLB (AWS) |
| **External DNS** | Automatic DNS record sync |
| **External Secrets Operator** | Cloud secrets → Kubernetes Secrets |
| **Karpenter** | Just-in-time node provisioning (AWS) |
| **Metrics Server** | CPU/memory metrics for HPA |
| **Storage Class** | GP3 EBS / Persistent Disk / Managed Disk |

## Live Updates

The Clusters page polls every 5 seconds while the tab is focused (TanStack Query pauses the interval in the background) — the interim reconciliation cadence. When a project's default environment reaches `ACTIVE` (deploy completes), the next poll surfaces the new cluster.

## Data Source

Cluster information comes from the DEPLOY job's `finalizeDeployment()` server action, which extracts OpenTofu outputs (cluster name, endpoint, ARN, ArgoCD URL, database/cache endpoints) and writes them to the project's component tables.
