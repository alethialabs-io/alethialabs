# Secrets

The secrets section creates entries in your cloud provider's secret management service. Secrets can be auto-generated with configurable parameters or set to custom values after provisioning.

## Fields Per Secret

| Field | Type | Description |
|-------|------|-------------|
| **Name** | Text input | Secret name (alphanumeric + hyphens only) |
| **Provider** | Select | Cloud-native store or a [pluggable connector](/console/connectors/pluggable) |
| **Generate** | Toggle | Auto-generate a random value |
| **Length** | Number | 8–128 characters |
| **Special Characters** | Toggle | Include `!@#$%^&*()` in generated value |

## Secrets Provider

By default secrets are created in the cluster cloud's native secret store. Select **HashiCorp Vault** to manage them in a Vault instance instead:

- Requires a connected [Vault connector](/console/connectors/pluggable) (Vault address + token).
- Per-Spec knobs: the KV **mount path** and **KV version** (1 or 2).
- The cloud-native secret resources are skipped at provision time; Vault entries are managed by the composed module.

## Presets

Quick-add buttons create secrets with recommended settings:

| Preset | Length | Special Chars | Use Case |
|--------|--------|--------------|----------|
| PostgreSQL Password | 32 | Yes | Database master password |
| API Token | 48 | No | Application API key |
| JWT Secret | 64 | No | Token signing key |
| Redis Password | 32 | No | Cache authentication |
| Custom Secret | 32 | No | User-defined |

## Integration with Kubernetes

After provisioning, secrets are synced to Kubernetes via the **External Secrets Operator** (installed by ArgoCD — see [GitOps & ArgoCD](/concepts/gitops-argocd)). This creates Kubernetes `Secret` resources from cloud secret store entries, keeping them in sync automatically.
