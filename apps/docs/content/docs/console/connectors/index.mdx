# Connectors

The Connectors page manages connections to cloud providers (AWS, GCP, Azure), Git providers (GitHub, GitLab, Bitbucket), and **pluggable per-category providers** (DNS, secrets, registry, observability). Cloud and Git connections must be established before provisioning; pluggable connectors are optional — they let a Project use a tool you already run instead of the cloud-native default.

## How It Works

Connectors are displayed as cards organized by category:

| Category | Providers | Auth |
|----------|-----------|------|
| **Cloud Providers** | AWS, GCP, Azure | cloud federation (no stored keys) |
| **Git Providers** | GitHub, GitLab, Bitbucket | OAuth |
| **DNS** | Cloudflare | API key |
| **Secrets** | HashiCorp Vault | API key |
| **Registry** | Docker Hub | API key |
| **Observability** | Datadog, Grafana Cloud, Prometheus | API key |

Each card shows:
- Provider icon and name
- Connection status: **Connected** (green), **Needs Reconnection** (amber), or **Not Connected**
- Connected account identifier (AWS Account ID, GCP Project, GitHub username)
- Action button (Connect / Disconnect / Reconnect)

## Connection Status

| Status | Meaning |
|--------|---------|
| **Connected** | Credentials valid and verified |
| **Needs Reconnection** | Git token expired, automatic refresh failed |
| **Not Connected** | No credentials on file |
| **Coming Soon** | Provider supported but not yet implemented |

## Cloud Provider Connections

Cloud connections use the platform's [zero-credential model](/concepts/security) — no static keys are stored. Each provider uses a federation mechanism:

<Cards>
  <Card title="AWS" href="/console/connectors/aws">Cross-Account IAM Role with External ID</Card>
  <Card title="GCP" href="/console/connectors/gcp">Workload Identity Federation with OIDC</Card>
  <Card title="Azure" href="/console/connectors/azure">Federated Identity via Entra ID</Card>
</Cards>

After connecting, Alethia verifies the credentials **instantly, server-side** (an auth + provisioning-permission probe — no runner, no job) and marks the connection `Connected`, `Degraded`, or `Disconnected`. Existing resources (VPCs, subnets, DNS zones) are discovered + kept fresh by the server-side, event-driven asset inventory. See [Connection health & inventory](/console/connectors/health-and-inventory).

## Git Provider Connections

Git connections use standard OAuth. See [Git Providers](/console/connectors/git-providers) for details.

## Pluggable Connectors

DNS, secrets, registry, and observability providers authenticate with an **API key / token**. Secret fields are encrypted at rest (`ALETHIA_CRED_ENCRYPTION_KEY`) and only decrypted on the runner at provision time — never stored in a Project snapshot. After connecting, a verification ping confirms the credential works.

## Disconnecting

- **Cloud providers**: Removes the `cloud_identities` record. Existing projects referencing that identity can no longer run jobs.
- **Git providers**: Removes the Better Auth `account` record and revokes the OAuth grant.
- **Pluggable connectors**: Removes the `connector_credentials` record. Projects referencing that provider fall back to failing validation until reconnected.
