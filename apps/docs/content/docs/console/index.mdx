# Console

The Console (`apps/console`) is Alethia's web control plane. It provides a visual interface for configuring infrastructure, managing [cloud identities](/console/connectors), monitoring [jobs](/console/jobs), and viewing provisioned clusters — all from the browser. Alethia is the platform; the Console is the web app you drive it from.

## Tech Stack

| Layer | Technology |
|-------|-----------|
| Framework | Next.js 15 (App Router, Server Actions) |
| Database | PostgreSQL + Drizzle ORM with org-scoped Row Level Security |
| Auth | Better Auth (social OAuth: GitHub, Google native; GitLab, Bitbucket genericOAuth; email 6-digit OTP) |
| Realtime | Postgres LISTEN/NOTIFY → SSE ([how it works](/concepts/realtime)) |
| Storage | S3-compatible storage (default SeaweedFS) for OpenTofu state and plan artifacts |
| UI | Tailwind CSS, shadcn/ui, Zustand state management |

## Architecture

<Mermaid chart={`
flowchart TD
  B[Browser] --> A[Next.js App Router]
  A --> SA[Server Actions<br/>Drizzle · org-scoped RLS]
  A --> API[API Routes<br/>Runner + CLI token auth]
  A --> SSE[LISTEN/NOTIFY → SSE]
  SSE --> ST[Live updates → browser stores]
`} />

All database access goes through Drizzle with org-scoped RLS policies enforcing per-org data isolation through a Policy Decision Point (PDP). Server Actions set the current org context (`app.current_org` via `set_config`) automatically.

## State Management

The Console uses five Zustand stores for client-side state, each with a 30-second stale threshold and real-time event handlers:

| Store | Purpose |
|-------|---------|
| `useProjectStore` | Form pricing data during project creation |
| `useJobsStore` | Job list with filters, pagination, real-time upserts |
| `useRunnersStore` | Runner list with deploy/update/destroy actions |
| `useClustersStore` | Active clusters from provisioned projects |
| `useCloudProviderStore` | Selected provider, identity, cached resources |

Real-time subscriptions are established on dashboard layout mount and feed updates into these stores. See [Real-time Architecture](/concepts/realtime) for details.
