# Alethia Web Platform

Alethia is the web-based control plane. It provides a visual interface for configuring infrastructure, managing cloud integrations, monitoring jobs, and viewing provisioned clusters — all from the browser.

## Tech Stack

| Layer | Technology |
|-------|-----------|
| Framework | Next.js 15 (App Router, Server Actions) |
| Database | PostgreSQL + Drizzle ORM with org-scoped Row Level Security |
| Auth | Better Auth (OAuth: GitHub, GitLab, Bitbucket, Google; email OTP) |
| Realtime | Postgres LISTEN/NOTIFY → SSE ([how it works](/concepts/realtime)) |
| Storage | S3-compatible storage (default SeaweedFS) for Terraform state and plan artifacts |
| UI | Tailwind CSS, shadcn/ui, Zustand state management |

## Architecture

```
Browser ──► Next.js App Router
              ├── Server Actions (Drizzle with org-scoped RLS)
              ├── API Routes (Worker + CLI token auth)
              └── LISTEN/NOTIFY → SSE (Server-Sent Events)
                   └── Live updates → browser stores
```

All database access goes through Drizzle with org-scoped RLS policies enforcing per-org data isolation through a Policy Decision Point (PDP). Server Actions set the current org context automatically.

## State Management

Alethia uses six Zustand stores for client-side state, each with a 30-second stale threshold and real-time event handlers:

| Store | Purpose |
|-------|---------|
| `useSpecStore` | Form pricing data during spec creation |
| `useJobsStore` | Job list with filters, pagination, real-time upserts |
| `useRunnersStore` | Runner list with deploy/update/destroy actions |
| `useClustersStore` | Active clusters from provisioned specs |
| `useZonesStore` | Zone hierarchy with spec status updates |
| `useCloudProviderStore` | Selected provider, identity, cached resources |

Real-time subscriptions are established on dashboard layout mount and feed updates into these stores. See [Real-time Architecture](/concepts/realtime) for details.
