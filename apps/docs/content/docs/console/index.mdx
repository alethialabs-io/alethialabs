# Console

The Console (`apps/console`) is Alethia's web control plane. It provides a visual interface for configuring infrastructure, managing [cloud identities](/console/connectors), monitoring [jobs](/console/jobs), and viewing provisioned clusters — all from the browser. Alethia is the platform; the Console is the web app you drive it from.

## Tech Stack

| Layer | Technology |
|-------|-----------|
| Framework | Next.js 16 (App Router, Server Actions) |
| Database | PostgreSQL + Drizzle ORM with org-scoped Row Level Security |
| Auth | Better Auth (social OAuth: GitHub, Google native; GitLab, Bitbucket genericOAuth; email 6-digit OTP) |
| Realtime | Postgres LISTEN/NOTIFY → SSE ([how it works](/concepts/realtime)) |
| Storage | S3-compatible storage (default SeaweedFS) for OpenTofu state and plan artifacts |
| UI | Tailwind CSS, shadcn/ui, Zustand state management |

## Architecture

<Mermaid chart={`
flowchart TD
  B[Browser] --> A[Next.js App Router]
  A --> SA[Server Actions<br/>Drizzle · org-scoped RLS]
  A --> API[API Routes<br/>Runner + CLI token auth]
  A --> SSE[LISTEN/NOTIFY → SSE]
  SSE --> ST[Live job-log & message streams]
`} />

All database access goes through Drizzle with org-scoped RLS policies enforcing per-org data isolation through a Policy Decision Point (PDP). Server Actions set the current org context (`app.current_org` via `set_config`) automatically.

## State Management

Server data lives in **TanStack Query** (React Query) — projects, jobs, runners, clusters, and cloud inventory are fetched, cached, and revalidated there. Zustand holds only **client-side UI and ephemeral state**: filter selections, and form/canvas working state. Key stores:

| Store | Purpose |
|-------|---------|
| `useWorkspaceStore` | The active organization the session is scoped to (drives the PDP + RLS) |
| `useCloudProviderStore` | Selected provider, cloud identity, and cached cloud-resource inventory |
| `usePricingStore` | Region price data during project creation |
| `useProjectsStore` | Starred (favorite) projects on the overview |
| `useJobsFilters` / `useRunnerFilters` | Filter selections for the jobs and runners lists |
| `useCanvasStore` | The Design-a-Project architecture graph (nodes and edges) |
| `useElenchStore` | Elench assistant UI state |
| `useNotificationsStore` | Read-notification ids |

Real-time job logs and support messages stream over SSE (Postgres LISTEN/NOTIFY). See [Real-time Architecture](/concepts/realtime) for details.
