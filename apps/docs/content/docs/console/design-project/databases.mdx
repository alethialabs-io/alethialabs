# Databases

The databases section configures managed relational database instances. Multiple databases can be added per project, each with its own engine, version, and capacity settings.

## Fields Per Database

| Field | Type | Description |
|-------|------|-------------|
| **Name** | Text input | Unique per project, lowercase, used in resource naming |
| **Engine** | Select | Provider-specific engine (see below) |
| **Engine Version** | Auto-set | Determined by engine selection |
| **Min Capacity** | Number | Minimum capacity (ACUs for Aurora, vCPUs for Cloud SQL) |
| **Max Capacity** | Number | Maximum capacity for auto-scaling |
| **Port** | Number | Database port (default: 5432 for PostgreSQL, 3306 for MySQL) |
| **IAM Auth** | Toggle | Authenticate with short-lived cloud IAM tokens instead of a password. See [Keyless database auth](/console/design-project/keyless-database-auth) |

## Adding Multiple Databases

Click **"Add Database"** to add another instance. Each database gets its own card in the form. The first database defaults to:
- Name: `primary`
- Engine: Aurora PostgreSQL (or equivalent)
- Capacity: 0.5–4 ACUs

Common patterns:
- **Primary + Read Replica**: One write database + one read-optimized database
- **Multi-service**: Separate databases per microservice (e.g., `users-db`, `orders-db`)

## After Provisioning

Once deployed, the database card in the [project detail](/console/dashboard/project-detail) Services tab shows:
- **Writer endpoint** — primary connection string
- **Reader endpoint** — read replica connection (Aurora only)
- **Master credentials** — stored in Secrets Manager (ARN shown, copy button)
- **Status** — ACTIVE, CREATING, or FAILED

<Callout type="info">
**IAM authentication** lets a workload connect using its own cloud identity and a short-lived token
instead of a stored password — nothing to rotate, no secret in the cluster. A service bound to an
IAM-auth database picks this up automatically. See [Keyless database auth](/console/design-project/keyless-database-auth)
for how it works per cloud and its current rollout status.
</Callout>
