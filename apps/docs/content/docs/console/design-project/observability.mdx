# Observability

The observability section ships cluster metrics, logs, and traces to an external monitoring platform. Unlike DNS or secrets, observability has **no cloud-native default** — it is a connector-only component, so enabling it means choosing a provider and connecting its credential.

## Fields

| Field | Type | Description |
|-------|------|-------------|
| **Enabled** | Toggle | Master switch for observability |
| **Provider** | Select | Datadog or Grafana Cloud |
| **Provider options** | Varies | Provider-specific configuration (see below) |

## Providers

<Tabs items={["Datadog", "Grafana Cloud"]}>
  <Tab value="Datadog">
    Installs the **Datadog agent** via Helm. Requires a connected [Datadog connector](/console/connectors/pluggable) (API key + application key). Choose the **site** (e.g. `datadoghq.com`, `datadoghq.eu`).
  </Tab>
  <Tab value="Grafana Cloud">
    Installs the **Grafana Agent** remote-writing metrics to Grafana Cloud. Requires a connected [Grafana Cloud connector](/console/connectors/pluggable) (instance ID + API token) and the **remote-write URL**.
  </Tab>
</Tabs>

<Callout type="info">
Want a self-hosted metrics stack instead of an external platform? Add the **kube-prometheus-stack** [add-on](/cli/commands/addons) — it installs Prometheus, Grafana, and Alertmanager in-cluster. Prometheus is an add-on (an in-cluster workload), not an observability connector (a credential to an external system).
</Callout>

## Credentials

Provider credentials are connected on the [Connectors](/console/connectors/pluggable) page and encrypted at rest. They are decrypted and injected into the runner's OpenTofu variables at provision time — never stored in the Project snapshot.

<Callout type="info">
Observability is a **target-state** component — the connector backends and modules exist; the Project-designer selector is rolling out. Track the [Connectors Architecture](/concepts/connectors) page for the current contract.
</Callout>
