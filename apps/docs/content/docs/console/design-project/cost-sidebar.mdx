# Cost

The cost chip sits in the top-right of the Architecture canvas. It shows the environment's estimated monthly infrastructure cost, taken from the **last plan** — not a live client-side estimate.

For a deeper technical explanation, see [Cost Estimation](/concepts/cost-estimation).

## Where the number comes from

The runner runs [Infracost](/concepts/cost-estimation) on every **PLAN** and posts the cost breakdown back with the job. The chip reads the resulting monthly total (`monthlyCost`) off the environment's status — so the cost is only ever a real figure the platform computed, never a guess made in the browser.

The chip only appears once a project and environment exist (an existing environment on the Architecture route). In the design-only create flow there is nothing planned yet, so there is no chip.

## States

The chip is honest about what it knows:

| State | When | What it shows |
|-------|------|---------------|
| **Not priced** | The environment has never been planned (`monthlyCost` is null) | A dashed chip reading `Not priced`, tooltip "Run a plan to price this environment." |
| **Priced** | A plan has run | `Monthly $X.XX` (two decimals), tooltip "From the plan on `<timestamp>`" when the capture time is known |

A never-planned environment shows `Not priced` rather than `$0.00` — a fabricated zero is worse than an admitted unknown, because you would believe it.

## The full breakdown

The single headline number is the chip. The **per-resource breakdown** lives in the plan's artifact panel, under the Plan tab. The console parses the Infracost breakdown JSON (`parseCostBreakdown`) into a ranked list of cost-bearing resources:

- Each resource carries its `monthlyCost`, `hourlyCost`, `resourceType`, and any priced sub-resources.
- Free or cost-unknown resources (monthly cost `0` or absent) are skipped — the list only ranks priced ones.
- Resources are sorted by monthly cost, most expensive first.
- The plan also carries `totalMonthlyCost` and `totalHourlyCost`.

## Limitations

- The figure is only as fresh as the **last plan** — change the design and the number is stale until you plan again.
- Infracost prices what appears in the OpenTofu plan JSON. Usage-based resources (NoSQL, messaging, object storage) and data-transfer costs are not estimated up front.
- A malformed or partial breakdown degrades to "unknown cost" rather than surfacing a wrong number.
