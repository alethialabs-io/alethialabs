# Platform Overview

This platform is a hybrid multi-cloud infrastructure orchestration system. It combines a visual web interface, a command-line tool, and autonomous execution agents to provision production-grade Kubernetes clusters with databases, caches, networking, DNS, secrets, and GitOps — across AWS, GCP, and Azure.

<img src="/docs/diagrams/concepts/platform-overview.svg" alt="Platform Architecture" />

## Three-Component Model

| Component | Type | Language | Role |
|-----------|------|----------|------|
| **Console** | Web application | TypeScript (Next.js) | Visual configuration, job management, real-time monitoring |
| **alethia** | CLI tool | Go | Terminal-based operations, scripting, CI/CD integration |
| **Runner** | Execution agent | Go | Runs OpenTofu (`tofu`), installs ArgoCD, streams logs back to Console |

## Data Flow

<Mermaid chart={`
flowchart LR
  Web["Console (web form)"] --> Q["Job Queue (Postgres)"]
  CLI["alethia CLI"] --> Q
  Q -->|claim| Runner["Runner (agent)"]
  Runner -->|keyless assume| Cloud["Cloud Provider"]
  Runner -->|logs| Notify["LISTEN/NOTIFY"]
  Notify -->|SSE| Web
`} />

1. The user configures infrastructure through Console (web form) or the `alethia` CLI
2. Actions are queued as jobs in Postgres
3. A runner claims the job atomically and assumes temporary, keyless cloud credentials
4. The runner executes `tofu` and streams logs back in real time over LISTEN/NOTIFY → SSE
5. On completion, ArgoCD is installed for GitOps-managed deployments

## Multi-Cloud Support

Every infrastructure component has provider-specific implementations across three clouds:

| Capability | AWS | GCP | Azure |
|-----------|-----|-----|-------|
| Kubernetes | EKS | GKE | AKS |
| Relational DB | Aurora PostgreSQL/MySQL | Cloud SQL | Azure Database |
| Cache | ElastiCache (Redis/Valkey) | Memorystore | Azure Cache for Redis |
| NoSQL | DynamoDB | Firestore | Cosmos DB |
| Messaging | SQS + SNS | Pub/Sub | Service Bus |
| DNS | Route 53 | Cloud DNS | Azure DNS |
| Secrets | Secrets Manager | Secret Manager | Key Vault |
| Container Registry | ECR | Artifact Registry | ACR |
| Object Storage | S3 | Cloud Storage | Storage Account |
| WAF | AWS WAFv2 | Cloud Armor | Azure WAF |

## Tech Stack

**Web (Console):** Next.js 16, PostgreSQL + Drizzle ORM, Better Auth, LISTEN/NOTIFY + SSE realtime, S3-compatible storage (SeaweedFS), Tailwind CSS, shadcn/ui, Zustand

**CLI (alethia):** Go, Cobra, Charmbracelet (lipgloss, huh, bubbletea), Homebrew tap

**Agent (Runner):** Go single binary — on a managed Hetzner fleet VM or the customer's own cloud; bundles `tofu` + kubectl + Helm, **CLI-free** (no aws-cli / gcloud / azure-cli; keyless OIDC + in-process `runner kube-token`)

**Shared (core):** Go package — provisioner, OpenTofu wrapper, cloud provider abstraction, ArgoCD rendering, Infracost, Git, Helm, and the Elench `verify` + `drift` engines

**Infrastructure:** OpenTofu, S3-compatible storage / SeaweedFS (state), the in-app fleet scaler (Hetzner VMs), release-please

## Monorepo Structure

```
alethialabs/
├── apps/
│   ├── console/          # Next.js web platform
│   ├── cli/              # Go CLI tool
│   ├── runner/           # Go execution agent
│   └── docs/             # Documentation site (Fumadocs)
├── packages/
│   ├── core/             # Shared Go library (provisioner, verify, drift, argocd)
│   ├── @repo/ui/         # Shared shadcn/ui design system
│   └── @repo/*           # Shared web packages (brand, plan-catalog, email, config)
├── infra/
│   ├── templates/project/ # Infrastructure templates (AWS/GCP/Azure/Alibaba/Hetzner)
│   ├── templates/argocd/ # ArgoCD application templates
│   ├── templates/runner/ # Self-deployed runner template (ECS) — NOT the managed fleet
│   └── connector/        # Cloud account bootstrap (OIDC federation setup)
├── apps/console/lib/fleet/          # In-app managed-fleet scaler (Hetzner)
├── apps/console/lib/db/migrations/  # PostgreSQL schema migrations (Drizzle)
└── .github/workflows/    # CI/CD (release-please, release-cli, release-runner)
```
