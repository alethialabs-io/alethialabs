# Runner Agent

Runner is the execution engine — a single Go binary shipped as a container image. Runners claim
provisioning jobs from the queue, mint per-job cloud credentials keylessly, execute the `tofu`
binary (OpenTofu), install ArgoCD, and stream logs back in real time. The binary is **CLI-free**:
it bundles no `aws`/`gcloud`/`az`, and mints cluster tokens in-process via `runner kube-token` (see
[Cloud Credentials](/runner/cloud-credentials)).

## Key facts

| Property | Value |
|----------|-------|
| Language | Go (single binary) |
| IaC engine | OpenTofu (`tofu`) `v1.9.0`, baked into the image |
| Base image | Alpine 3.21, non-root `runner` user |
| Registries | GitHub Container Registry (`ghcr.io/alethialabs-io/runner*`) |
| Versioning | release-please (`runner-v*` tags). See [Release System](/concepts/release-system). |
| Poll interval | 10 seconds (`POST /api/jobs/claim`) |
| Heartbeat interval | 30 seconds (`POST /api/runners/heartbeat`) |
| tofu state | Console HTTP state proxy — no object-storage credential on the runner |
| Cloud auth | Keyless OIDC per job (no static keys) |

## Where a runner runs — the managed fleet vs. self-deployed

The word "runtime" splits cleanly into two paths. Do not conflate them.

| | Managed fleet | Self-deployed (AWS) |
|---|---|---|
| Operator | `managed` | `self` / `deployed` |
| Substrate | **Hetzner VM warm pool** | AWS ECS Fargate (ARM64) |
| Provisioned by | the in-app scaler (`apps/console/lib/fleet/`) | a `DEPLOY_RUNNER` job running `infra/templates/runner/aws/` |
| Image source | GHCR per-cloud image, self-registers via `ALETHIA_RUNNER_BOOTSTRAP_TOKEN` | GHCR image, registration token injected as an ECS env var |
| Scaling | 60s warm-pool loop, sized by queue depth ([Warm-pool scaling](/runner/scale-to-zero)) | ECS service `desired_count`, managed by the customer |
| Billing | provisioned hours via `runner_usage_sessions` | the customer's own AWS bill |

The **managed fleet** is Alethia running runners in the platform account and assuming into the
customer cloud — cheap Hetzner VMs from a GHCR image, sized by an in-app scaler. **ECS Fargate is
not the managed substrate anymore** (the retired AWS ECS + Lambda fleet lived under `infra/fleet-aws`
and is gone). ECS Fargate is real only for a **self-deployed** runner, which a `DEPLOY_RUNNER` job
provisions into the customer's own AWS account from `infra/templates/runner/aws/`. See
[Operator & provisioning](#operator--provisioning) and [Self-operated runners](/runner/self-hosted).

## Operator & provisioning

A runner is described by two orthogonal axes: **operator** (`managed` | `self`) and, for self
runners, **provisioning** (`deployed` | `registered`). This yields three effective categories:

| Category | Operator | Provisioning | Description |
|----------|----------|--------------|-------------|
| **Managed** | `managed` | — | Alethia runs the runner in the platform account (Hetzner VM warm pool) and bills it by provisioned hours. Automatic scaling, no user setup. |
| **Self · Deployed** | `self` | `deployed` | Provisioned into the customer's cloud by an existing runner running `tofu` (Console "Deploy" tab / `alethia runner deploy`). AWS lands on ECS Fargate. |
| **Self · Registered** | `self` | `registered` | Customer brings their own runner — their own IaC or just `alethia runner start` — and registers it for a token. For on-prem or air-gapped environments. |

The legacy `ALETHIA_RUNNER_MODE` (`cloud-hosted`/`self-hosted`) still works as a back-compat
fallback (`cloud-hosted` → `managed`, `self-hosted` → `self`), but `ALETHIA_RUNNER_OPERATOR` is
canonical.

## What runner does

<Steps>

### Polls for jobs
Every 10 seconds, calls `POST /api/jobs/claim`. See [Job Queue Pattern](/concepts/job-queue).

### Claims atomically
PostgreSQL `claim_next_job()` with `FOR UPDATE SKIP LOCKED` prevents double-execution.

### Mints credentials
Keyless OIDC per job — AWS `AssumeRoleWithWebIdentity`, GCP Workload Identity Federation, Azure
federated identity, Alibaba `AssumeRoleWithOIDC` — no stored cloud keys. Token clouds
(DigitalOcean / Hetzner / Civo) use an API token, either stored encrypted or kept entirely in a
self-operated runner. See [Cloud Credentials](/runner/cloud-credentials).

### Executes OpenTofu
`tofu init` → `tofu plan` → the [verify gate](/runner/job-execution/plan-jobs) → `tofu apply` or
`tofu destroy`. Provisions **greenfield** infrastructure, or **attaches to existing** resources when
the config marks a component "existing" (brownfield — e.g. an existing VPC/VNet). See
[Job Execution](/runner/job-execution).

### Installs ArgoCD
Helm chart deployment with the App-of-Apps pattern. See [GitOps & ArgoCD](/concepts/gitops-argocd).

### Streams logs
Batched POST every 2 seconds → Postgres `LISTEN/NOTIFY` → SSE → browser. See
[Log Streaming](/runner/log-streaming).

### Reports results
Updates job status with execution metadata (timing, tofu outputs, and the verify report on
`execution_metadata["verify_result"]`).

### Sends heartbeat
Every 30 seconds to confirm it is alive.

</Steps>

## Container contents

The image builds `FROM` a shared `runner-base` (Alpine 3.21, non-root `runner` user). The base
carries everything cloud-agnostic; each per-cloud image adds only that cloud's project template.

| Tool | Purpose |
|------|---------|
| Go binary (`runner`) | The runner executable — also the `kube-token` exec-credential plugin |
| OpenTofu (`tofu`) `v1.9.0` | Infrastructure provisioning (SHA256-verified download, baked in) |
| kubectl | Kubernetes cluster access |
| Helm | ArgoCD and component installation |
| Infracost `v0.10.39` | Cost estimation |
| Infrastructure templates | Per-cloud project templates + runner + ArgoCD manifests |
| podman / crun / fuse-overlayfs | Nested rootless sandbox runtime — **inert** until `ALETHIA_SANDBOX_BACKEND=container` |

<Callout type="info">
  **No cloud CLIs.** The image bundles **no** `aws-cli`, `gcloud`, or `azure-cli` — the
  `Dockerfile`s say so explicitly. EKS/GKE/AKS cluster tokens are minted **in-process** by
  `runner kube-token --provider <cloud>`, invoked by the exec-credential kubeconfig the runner
  writes. See [Cloud Credentials](/runner/cloud-credentials).
</Callout>
