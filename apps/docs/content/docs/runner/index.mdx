# Runner Agent

Runner is the execution engine. It's a Go binary packaged as a Docker container that runs in AWS ECS Fargate (ARM64). Runners claim jobs from the queue, assume temporary cloud credentials, execute Terraform, install ArgoCD, and stream logs back in real-time.

## Key Facts

| Property | Value |
|----------|-------|
| Language | Go |
| Runtime | ECS Fargate (ARM64 / Graviton) |
| Registries | AWS ECR + GitHub Container Registry |
| Versioning | release-please (`runner-v*` tags). See [Release System](/concepts/release-system). |
| Poll interval | 10 seconds |
| Heartbeat interval | 30 seconds |
| Idle scale-down | After 5 minutes ([Scale-to-Zero](/runner/scale-to-zero)) |

## Operator & Provisioning

A Runner is described by two orthogonal fields: **operator** (`managed` | `self`) and, for self runners, **provisioning** (`deployed` | `registered`). This yields three effective categories:

| Category | Operator | Provisioning | Description |
|----------|----------|--------------|-------------|
| **Managed** | `managed` | — | Alethia operates the runner in our own account (ECS Fargate) and bills it by provisioned hours. Automatic scaling, no user setup. |
| **Self · Deployed** | `self` | `deployed` | Provisioned into the customer's cloud account by an existing runner running Terraform (console "Deploy" tab / `alethia runner deploy`). |
| **Self · Registered** | `self` | `registered` | Customer brings their own runner — their own Terraform or just `alethia runner start` — and registers it for a token. For on-prem or air-gapped environments. |

## What Runner Does

<Steps>

### Polls for jobs
Every 10 seconds, calls `POST /api/jobs/claim`. See [Job Queue Pattern](/concepts/job-queue).

### Claims atomically
PostgreSQL `claim_next_job()` with `FOR UPDATE SKIP LOCKED` prevents double-execution.

### Assumes credentials
STS AssumeRole (AWS), RAM AssumeRole (Alibaba), WIF token exchange (GCP), or OIDC federated identity (Azure) — all zero stored credentials. Token clouds (DigitalOcean / Hetzner / Civo) use an API token, either stored encrypted or kept entirely in your own self-hosted runner. See [Cloud Credentials](/runner/cloud-credentials).

### Executes Terraform
`init` → `plan` → `apply` or `destroy`. Provisions **greenfield** infrastructure, or **attaches to
existing** resources when the config marks a component "existing" (brownfield — e.g. deploy into an
existing VPC/VNet). See [Job Execution](/runner/job-execution).

### Installs ArgoCD
Helm chart deployment with App of Apps pattern. See [GitOps & ArgoCD](/concepts/gitops-argocd).

### Streams logs
Batched POST every 2 seconds → Postgres LISTEN/NOTIFY → SSE → browser. See [Log Streaming](/runner/log-streaming).

### Reports results
Updates job status with execution metadata, timing, and Terraform outputs.

### Sends heartbeat
Every 30 seconds to confirm it is alive.

</Steps>

## Container Contents

The Docker image (Alpine 3.21, ARM64, non-root user) includes:

| Tool | Purpose |
|------|---------|
| Go binary | The Runner executable |
| Terraform 1.7.4 | Infrastructure provisioning |
| kubectl | Kubernetes cluster access |
| Helm | ArgoCD and component installation |
| AWS CLI | AWS credential handling |
| Google Cloud SDK | GCP credential handling |
| Azure CLI | Azure credential handling |
| Infracost | Cost estimation |
| Infrastructure templates | AWS, GCP, Azure Terraform modules + ArgoCD manifests |
