# Real-Time Log Streaming

The runner streams all `tofu` output back to Alethia in real time. Users see `tofu plan` and
`tofu apply` output in the browser as it happens.

## Architecture

<Mermaid chart={`
flowchart TD
  Tofu[tofu process] --> Out[JobLogger STDOUT]
  Tofu --> Err[JobLogger STDERR]
  Out --> Buf[Buffer]
  Err --> Buf
  Buf -->|POST /api/jobs/id/logs| Tbl[(provision_job_logs)]
  Tbl -->|NOTIFY on INSERT| Listen[Console LISTEN]
  Listen -->|SSE stream| Browser[Browser EventSource]
`} />

## JobLogger

The `JobLogger` is a Go struct that wraps an `io.Writer`. It buffers output and flushes to the
Alethia API periodically.

**Two loggers per job:**
- **STDOUT logger** — captures normal `tofu` output (plan details, resource progress)
- **STDERR logger** — captures errors and warnings

**Buffer flush triggers:**
- Every **2 seconds** (ticker-based flush)
- When the buffer exceeds **10 KB** (size-based flush)
- On job completion (final flush)

## API Delivery

Each flush sends a batch POST:

```
POST /api/jobs/{job_id}/logs
Content-Type: application/json

{
  "log_chunk": "Creating VPC...\nVPC created: vpc-12345\n",
  "stream_type": "STDOUT"
}
```

The Alethia API inserts the chunk into `provision_job_logs`:

```sql
INSERT INTO provision_job_logs (job_id, log_chunk, stream_type, created_at)
VALUES ($1, $2, $3, now());
```

## LISTEN/NOTIFY → SSE

An `AFTER INSERT` trigger on `provision_job_logs` fires a Postgres `NOTIFY` carrying the new chunk. The console holds a single `LISTEN` connection per app instance, receives the notification, and relays it to the browser over an SSE stream scoped to the `job_id`. The browser consumes that stream with an `EventSource`:

```typescript
const events = new EventSource(`/api/jobs/${jobId}/logs/stream`);

events.addEventListener('log', (event) => {
  const { log_chunk, stream_type } = JSON.parse(event.data);
  appendLogChunk(log_chunk, stream_type);
});
```

Each new batch appears in the log viewer within milliseconds of being written by the Runner. (For multi-instance deployments, an optional Redis transport relays notifications between console instances — enterprise.)

## Log Display

The job detail page renders logs in a terminal-like viewer:

- **STDOUT** lines in default color (white/light gray)
- **STDERR** lines in red
- **SYSTEM** lines in blue (platform messages like "Minting keyless credentials..." or "Starting tofu plan...")
- Auto-scroll to bottom as new lines arrive
- Manual scroll lock when the user scrolls up

## Local Echo

All log output is also written to the container's stdout — journald/`docker logs` on a managed
Hetzner fleet VM, CloudWatch Logs for a self-deployed runner on ECS Fargate, or the local terminal
for a registered runner. This provides a secondary log source for debugging.

## Timing

| Event | Latency |
|-------|---------|
| tofu writes to stdout | 0ms (piped to JobLogger) |
| JobLogger buffer flush | ≤ 2 seconds (ticker interval) |
| API POST to Alethia | ~50ms (network) |
| Database INSERT | ~5ms |
| NOTIFY → SSE delivery | ~50ms (server-sent event) |
| **Total: tofu output → browser** | **~2-3 seconds** |
