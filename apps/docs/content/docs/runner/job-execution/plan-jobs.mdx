# Plan Jobs

PLAN jobs run `terraform plan` to preview infrastructure changes without applying them.

## Execution Flow

<Steps>

### Assume cloud credentials
Provider-specific: STS AssumeRole (AWS), WIF exchange (GCP), or federated identity (Azure). See [Cloud Credentials](/runner/cloud-credentials).

### Generate Terraform variables
The spec's config snapshot is transformed into `terraform.tfvars` by the provisioner. Each spec component maps to specific Terraform variables.

### terraform init
Initializes with the [Supabase S3 backend](/concepts/terraform-state):
```
bucket = "spec-terraform-state"
key    = "{zone}/{project}/{stage}/{region}/terraform.tfstate"
```

### terraform plan
```bash
terraform plan -out=plan.tfplan
```
Generates a binary plan file describing what will change.

### terraform show
```bash
terraform show -json plan.tfplan
```
Converts the binary plan to structured JSON output for UI display.

### Infracost analysis (optional)
If `INFRACOST_API_KEY` is configured:
```bash
infracost breakdown --path plan.json --format json
```
Parses per-resource cost estimates. See [Cost Estimation](/concepts/cost-estimation).

### Upload plan artifact
The binary `plan.tfplan` file is uploaded to Supabase Storage (`plan-artifacts` bucket) for later use by DEPLOY jobs.

### Report results
Job status → SUCCESS with execution metadata:
- Resource change counts (add / change / destroy)
- Cost breakdown (if Infracost ran)
- Plan artifact reference

</Steps>

## What the User Sees

After the plan completes, the [Plan tab](/console/jobs/plan-and-apply) in the spec detail page shows:
- Resource groups with create/update/delete actions
- Per-resource costs
- Total estimated monthly cost
- An "Apply" button to proceed
