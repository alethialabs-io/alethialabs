# Job Execution

All jobs share the same claiming mechanism and keyless credential handling, but differ in what they
execute. For the queue mechanics, see [Job Queue Pattern](/concepts/job-queue).

## Claiming

Every 10 seconds the runner calls `POST /api/jobs/claim`. The server runs `claim_next_job()` — a
PostgreSQL function using `FOR UPDATE SKIP LOCKED` for atomic, exactly-once claiming. Job status
walks `QUEUED → CLAIMED → PROCESSING → SUCCESS | FAILED | CANCELLED`.

When a job is returned, the runner:
1. Sets status to `PROCESSING`.
2. Reads the environment config snapshot and cloud identity from the claim.
3. Mints cloud credentials keylessly ([details](/runner/cloud-credentials)).
4. Dispatches on `types.JobType` (the switch in `internal/agent/runner.go`) to the handler.
5. Sets status to `SUCCESS` or `FAILED`.

## Job types

The dispatch switch routes each of these `provisionJobType` values:

| Job type | Handler | Does |
|----------|---------|------|
| `PLAN` | `executePlan` | `tofu plan` + verify gate + Infracost → plan artifact. [Details](/runner/job-execution/plan-jobs) |
| `DEPLOY` | `executeDeploy` | verify gate (fail-closed) → `tofu apply` → keyless kubeconfig → ArgoCD. [Details](/runner/job-execution/deploy-jobs) |
| `DESTROY` | `executeDestroy` | `tofu destroy`. [Details](/runner/job-execution/destroy-jobs) |
| `DEPLOY_RUNNER` / `UPDATE_RUNNER` / `DESTROY_RUNNER` | `executeDeployRunner` / … | Provision, update, or tear down a self-deployed runner. [Details](/runner/job-execution/runner-jobs) |
| `ANALYZE_REPO` | `executeAnalyzeRepo` | Clone + statically scan a repo into a `RepoDigest`. |
| `DETECT_DRIFT` | `executeDriftDetection` | `tofu plan -refresh-only` → `drift.Analyze` → post a `Posture`. |
| `AUDIT` | `executeAudit` | Run the Elench verify engine over a bring-your-own plan or k8s manifests → post `verify_result`. Provisions nothing. |
| `CHART_SCAN` | `executeChartScan` | Render a BYO Helm chart to manifests + scan it (zero secrets, deny-all egress) — always sandboxed. |
| `IAC_SCAN` | `executeIacScan` | Static scan of bring-your-own IaC. |
| `STATE_SURGERY` | `executeStateSurgery` | Operator-driven tofu state repair. |

## Error handling

When any step fails:
1. The error is logged to the STDERR stream.
2. Job status is set to `FAILED` with the error message.
3. tofu state is preserved through the [Console state proxy](/concepts/terraform-state) (partial
   apply state, with fenced locks so a reclaimed lock can't clobber a slow writer).
4. Cloud credentials are cleared (the deferred cleanup always runs).
5. The runner returns to the poll loop for the next job.

Users can retry failed jobs — since state is preserved, retries continue from the last successful
resource rather than starting over.
