# Deploy Jobs

DEPLOY jobs apply the Terraform plan and install ArgoCD on the provisioned cluster.

## Execution Flow

<Steps>

### Assume cloud credentials
Same as PLAN jobs.

### Download plan artifact
Fetches the binary `plan.tfplan` from the PLAN job's S3-compatible storage upload.

### Validate config hash
Compares the project's current config hash against the plan's config hash. If they differ (project was modified after planning), the job fails:

> "Configuration has changed since the plan was generated. Please re-plan."

This prevents applying a stale plan. See [Plan & Apply](/console/jobs/plan-and-apply).

### terraform init
Initializes with the S3 backend (same as PLAN).

### terraform apply
```bash
terraform apply plan.tfplan
```
Provisions all infrastructure resources. State is written back to S3 after each resource operation.

### Extract Terraform outputs
Reads outputs: cluster name, endpoint, ARN, database endpoints, cache endpoints, VPC ID, etc.

### Generate kubeconfig
Calls the provider-specific CLI:

<Tabs items={["AWS", "GCP", "Azure"]}>
  <Tab value="AWS">
    ```bash
    aws eks update-kubeconfig --name {cluster} --region {region}
    ```
  </Tab>
  <Tab value="GCP">
    ```bash
    gcloud container clusters get-credentials {cluster} --region {region}
    ```
  </Tab>
  <Tab value="Azure">
    ```bash
    az aks get-credentials --name {cluster} --resource-group {rg}
    ```
  </Tab>
</Tabs>

### Install ArgoCD
Deploys ArgoCD via Helm chart, waits for pods to be Ready, extracts admin password. See [GitOps & ArgoCD](/concepts/gitops-argocd).

### Configure App of Apps
Renders ArgoCD Application manifests with infrastructure facts (cluster name, DB endpoints, etc.) as Helm values. Applies manifests to the cluster.

### Report results
Job status → SUCCESS with execution metadata:
- Cluster name, endpoint, ARN
- ArgoCD URL and admin password
- All Terraform outputs
- Execution timing per phase

</Steps>

## Finalization

On the Alethia side, when the job reaches SUCCESS, the `finalizeDeployment()` server action:
- Updates `project_cluster` with cluster metadata and ArgoCD credentials
- Updates `project_databases` with writer/reader endpoints and credential ARNs
- Updates `project_caches` with primary/reader endpoints
- Sets project status to `ACTIVE`

This data appears in the [Clusters page](/console/clusters) and project detail.
