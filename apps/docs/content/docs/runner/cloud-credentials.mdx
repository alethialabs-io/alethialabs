# Cloud Credential Management

Runner assumes temporary cloud credentials at the start of every job and clears them when the job completes. No credentials persist between jobs.

## Per-Job Credential Lifecycle

```
Job start
  └── Read cloud_identity from job configuration
       └── Provider-specific credential assumption
            └── Set environment variables
                 └── Execute Terraform (uses env vars)
                      └── defer: Clear credentials
```

The `defer` ensures credentials are always cleared, even if the job fails or panics.

## AWS: STS AssumeRole

**Input from cloud_identity:**
- `role_arn` — ARN of the IAM role in the user's AWS account
- `external_id` — shared secret for confused deputy prevention

**Execution:**

```go
func AssumeRole(roleArn, externalId string) error {
    stsClient := sts.NewFromConfig(cfg)
    result, err := stsClient.AssumeRole(ctx, &sts.AssumeRoleInput{
        RoleArn:         &roleArn,
        ExternalId:      &externalId,
        RoleSessionName: aws.String("alethia-runner"),
        DurationSeconds: aws.Int32(3600), // 1 hour
    })
    // Set environment variables
    os.Setenv("AWS_ACCESS_KEY_ID", *result.Credentials.AccessKeyId)
    os.Setenv("AWS_SECRET_ACCESS_KEY", *result.Credentials.SecretAccessKey)
    os.Setenv("AWS_SESSION_TOKEN", *result.Credentials.SessionToken)
    return nil
}
```

**Cleanup:**

```go
defer func() {
    os.Unsetenv("AWS_ACCESS_KEY_ID")
    os.Unsetenv("AWS_SECRET_ACCESS_KEY")
    os.Unsetenv("AWS_SESSION_TOKEN")
}()
```

Terraform reads these environment variables automatically via the AWS provider.

## Alibaba Cloud: RAM Role (Zero Stored)

Alibaba mirrors the AWS model. Runner assumes the customer's **RAM role** with an `ExternalId`
(confused-deputy guard) using Alethia's *platform* RAM key — `ALETHIA_ALIBABA_ACCESS_KEY_ID` /
`ALETHIA_ALIBABA_ACCESS_KEY_SECRET`, one platform secret, never the customer's — and exports temporary
`ALICLOUD_*` credentials. The customer stores **zero** credentials: only the role ARN + a stable ExternalId.

**Input from cloud_identity:**
- `role_arn` — ARN of the RAM role in the user's account (`acs:ram::<account>:role/<name>`)
- `external_id` — shared secret for confused-deputy prevention

**Execution:**

```go
func ActivateAlibabaRole(roleArn, externalId, sessionName string) (cleanup func(), err error) {
    // STS AssumeRole using Alethia's PLATFORM RAM key (never the customer's).
    result, err := stsClient.AssumeRole(&sts.AssumeRoleRequest{
        RoleArn:         &roleArn,
        ExternalId:      &externalId,
        RoleSessionName: &sessionName,
    })
    os.Setenv("ALICLOUD_ACCESS_KEY", result.Credentials.AccessKeyId)
    os.Setenv("ALICLOUD_SECRET_KEY", result.Credentials.AccessKeySecret)
    os.Setenv("ALICLOUD_SECURITY_TOKEN", result.Credentials.SecurityToken)
    return cleanup, nil // cleanup unsets the ALICLOUD_* vars
}
```

Terraform's `alicloud` provider reads these automatically. The runner needs Alethia's platform RAM key
configured; without it the job fails clearly (`platform Alibaba credentials not configured`).

## GCP: Workload Identity Federation

**Input from cloud_identity:**
- `project_id` — GCP project ID
- `wif_config` — Workload Identity Federation pool and provider configuration

**Execution:**

```go
func ActivateGcpWIF(projectId string, wifConfig WIFConfig) (cleanup func(), err error) {
    // Write credential configuration file
    credFile := filepath.Join(tmpDir, "gcp-wif-credential.json")
    writeCredentialConfig(credFile, wifConfig)

    // Set environment variable
    os.Setenv("GOOGLE_APPLICATION_CREDENTIALS", credFile)
    os.Setenv("GOOGLE_PROJECT", projectId)

    return func() {
        os.Remove(credFile)
        os.Unsetenv("GOOGLE_APPLICATION_CREDENTIALS")
        os.Unsetenv("GOOGLE_PROJECT")
    }, nil
}
```

The credential configuration file tells the GCP SDK to exchange an OIDC token for temporary GCP credentials via the STS endpoint. No service account key file is involved.

## Azure: Federated Identity

Azure is **keyless** — no client secret reaches the runner. The runner authenticates as Alethia's
platform Entra app by fetching a short-lived OIDC assertion from the console (the console holds the
issuer signing key) and handing it to OpenTofu.

**Input from cloud_identity:**
- `tenant_id` — the customer's Azure Entra tenant
- `client_id` — Alethia's platform app id (the same for every connection)
- `subscription_id` — Target Azure subscription

**Execution:**

```go
func ActivateAzureFederated(fetcher azureTokenFetcher, tenantId, clientId, subscriptionId string) (cleanup func(), err error) {
    // Mint an assertion from the console (POST /api/runners/azure-token, runner-authed).
    token, err := fetcher.FetchAzureToken()
    if err != nil {
        return nil, err
    }
    tokenFile := writeToTempFile(token)

    // azurerm/azuread (OpenTofu) read ARM_*; the Azure SDK + kubelogin read AZURE_*.
    os.Setenv("ARM_USE_OIDC", "true")
    os.Setenv("ARM_CLIENT_ID", clientId)
    os.Setenv("ARM_TENANT_ID", tenantId)
    os.Setenv("ARM_SUBSCRIPTION_ID", subscriptionId)
    os.Setenv("ARM_OIDC_TOKEN", token)
    os.Setenv("AZURE_FEDERATED_TOKEN_FILE", tokenFile)
    // …

    return cleanup, nil
}
```

The azurerm provider exchanges the ≤10-minute assertion for a ~1-hour ARM access token at provider
init, so a long apply rides that token — the short assertion TTL only needs to cover apply start.

## Token Clouds: DigitalOcean / Hetzner / Civo

These providers have no cross-account role federation — they authenticate with a single API token.
Alethia supports two models.

### Stored scoped token (encrypted)

The customer pastes a scoped API token; Alethia stores it **encrypted at rest** and decrypts it only at
claim time, passing it to the runner. The runner exports the provider's env var
(`DIGITALOCEAN_ACCESS_TOKEN` / `HCLOUD_TOKEN` / `CIVO_TOKEN`) for the job and clears it after.

### Self-managed — token stays in your infrastructure (true zero-trust)

For clouds without federation, the honest zero-trust path: the token lives **only** in your self-hosted
runner's environment, never in Alethia's database.

- The connection is recorded with `self_managed: true` and **no token**.
- Its jobs are flagged `requires_self_runner`, and `claim_next_job` **excludes managed runners** from
  them — so only *your* runner, which holds `HCLOUD_TOKEN` / `CIVO_TOKEN` / `DIGITALOCEAN_ACCESS_TOKEN`
  in its own env, can claim and run them.
- The runner reads the token from its environment (the `selfManaged` path) and never receives it from
  Alethia.

Pick the model in the connect sheet: **"Store a scoped token"** (encrypted at rest) vs **"Self-hosted
runner — token stays in your infra."** See [Self-Operated Runners](/docs/runner/self-hosted).

## Security Properties

| Property | Guarantee |
|----------|-----------|
| No persistent credentials | Env vars are cleared after every job via `defer` |
| No credential sharing | Each job gets its own assumed session |
| Time-bounded | AWS sessions expire after 1 hour |
| Scoped | Credentials only have permissions defined in the IAM role / service account / app registration |
| Revocable | User deletes the IAM role / WIF config / App Registration → instant revocation |
| No file leaks | Temporary credential files are deleted in `defer` cleanup |
| Optional zero-storage | Self-managed token clouds keep the secret in your runner's env — it never enters Alethia's database |

## CloudIdentity Structure

The `cloud_identities` table stores provider-specific credentials in a JSONB column:

**AWS:**
```json
{
  "role_arn": "arn:aws:iam::123456789012:role/AlethiaProvisionerRole",
  "external_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890"
}
```

**Alibaba:**
```json
{
  "role_arn": "acs:ram::123456789012:role/AlethiaProvisionerRole",
  "external_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890"
}
```

**GCP:**
```json
{
  "project_id": "my-gcp-project",
  "wif_config": {
    "pool_id": "alethia-pool",
    "provider_id": "alethia-provider",
    "service_account_email": "alethia@my-gcp-project.iam.gserviceaccount.com"
  }
}
```

**Azure:**
```json
{
  "tenant_id": "12345678-1234-1234-1234-123456789012",
  "client_id": "abcdefgh-abcd-abcd-abcd-abcdefghijkl",
  "subscription_id": "87654321-4321-4321-4321-210987654321"
}
```

**Token clouds (DigitalOcean / Hetzner / Civo):**
```json
// Stored scoped token — encrypted at rest
{ "token": { "ciphertext": "…", "iv": "…" } }

// Self-managed — the token lives in the runner's env, not here
{ "self_managed": true, "token": null }
```
