# Configuration

alethia stores configuration and credentials under the OS user-config directory (`~/.config/alethia`
on Linux, `~/Library/Application Support/alethia` on macOS — the location follows Go's
`os.UserConfigDir`). Inspect the resolved values and on-disk paths at any time with
[`alethia config`](#alethia-config).

## File Locations

| File | Purpose |
|------|---------|
| `~/.config/alethia/credentials.json` | CLI authentication tokens (access + refresh JWT) |
| `~/.config/alethia/config.json` | Active organization context (id, name, slug) |

## credentials.json

Created by `alethia login`. Contains JWT tokens for API authentication:

```json
{
  "access_token": "eyJ...",
  "refresh_token": "eyJ...",
  "user_email": "you@example.com"
}
```

The access token is refreshed automatically before each API call if expired. See [Authentication](/cli/authentication).

## config.json

Created/updated by `alethia org switch`. Records the active organization, which is sent as the
`X-Alethia-Org` header on every request:

```json
{
  "active_org_id": "org_123",
  "active_org_name": "Acme",
  "active_org_slug": "acme"
}
```

Clear it with `alethia config clear-context`.

## `alethia config`

Show the resolved CLI configuration: the control-plane URL (`ALETHIA_WEB_ORIGIN`), the active
organization context, and the on-disk paths of the credentials and config files.

```bash
alethia config
```

```
Field          Value
web origin     https://app.alethialabs.io (config)
active org     Acme
active org id  org_123
credentials    /Users/you/Library/Application Support/alethia/credentials.json
config         /Users/you/Library/Application Support/alethia/config.json
```

`alethia config clear-context` clears the active organization context. `alethia config -o json`
emits the same values as a JSON object.

## `alethia config export`

Export a project's resolved configuration. By default the configuration **content** is written to
stdout; pass `--out` to write it to a file, or `-o json` to get the export envelope (content +
filename + format).

```bash
# Print a project's configuration to stdout
alethia config export my-project

# Write it to a file
alethia config export my-project --out my-project.yaml

# Get the export envelope (content, filename, format) as JSON
alethia config export my-project -o json
```

| Flag | Default | Description |
|------|---------|-------------|
| `--format` | `legacy-yaml` | The configuration **content** format to request. |
| `--out` | _(stdout)_ | Write the content to this file instead of stdout. |

The project name is optional — omit it in an interactive terminal to pick one from a list. Backing
call: `GET /cli/configurations/by-project-name/<name>/export`.

## Runner Credentials

The `alethia` CLI does not store Runner credentials. A self-operated Runner is a
separate binary configured entirely through environment variables — register the
Runner in the console (or via the API) to obtain a `runner_id` / `runner_token`,
then set `ALETHIA_RUNNER_ID` and `ALETHIA_RUNNER_TOKEN` on the host. See
[Self-Operated Runners](/runner/self-hosted) for the full env contract.

## Environment Variables

| Variable | Default | Purpose |
|----------|---------|---------|
| `ALETHIA_WEB_ORIGIN` | `https://alethialabs.io` (compiled-in default) | Override the Alethia API base URL |

This is the only environment variable alethia uses. All other configuration comes from the JSON files above or command-line flags.

## Global Flags

Two persistent flags are available on every command:

| Flag | Default | Description |
|------|---------|-------------|
| `-o, --output <table\|json\|csv>` | `table` | Output format. |
| `--no-input` | off | Disable interactive prompts (fail instead of prompting). Auto-enabled when stdin is not a TTY. |

See [Identity & Global Options](/cli/identity) for output-format examples and scripting tips.

## Control-Plane URL Resolution

The control-plane URL resolves in precedence order: the `ALETHIA_WEB_ORIGIN` environment
variable, then the persisted config (`alethia config set web-origin <url>`), then a compiled-in
default (`https://alethialabs.io`). The hosted default is a constant baked into the binary, so the
production build needs no configuration (GoReleaser's `ldflags` inject only the CLI version, not the
URL). For development or a self-hosted control plane, override it with the environment variable:

```bash
export ALETHIA_WEB_ORIGIN=http://localhost:3000
alethia login  # authenticates against local server
```
