# Command Reference

alethia provides commands across 8 groups. All commands communicate with the Alethia API — they do not execute Terraform directly.

## Command Tree

```

├── login [--force]
├── logout
├── connector
│   ├── gcp [--project <id>] [--manual]
│   ├── aws [--region <r>] [--role-name <name>] [--manual]
│   ├── azure [--subscription <id>] [--manual]
│   ├── list
│   └── remove [provider] [--yes]
├── zone
│   ├── create --name <name>
│   ├── list

├── spec
```

## Interactive Selection

When `--spec-id` is omitted from `plan`, `apply`, or `destroy`, alethia launches an interactive TUI selector. See [TUI Components](/cli/tui) for how this works.

## Common Patterns

### Connect a Cloud Account

```bash
# Connect a GCP project (runs the installer in your Cloud Shell)
alethia connector gcp --project my-gcp-project

# List connected accounts
alethia connector list
```

See [Connector Commands](/cli/commands/connector) for AWS and Azure flows.

### Plan-Review-Apply

```bash
# 1. Plan
alethia plan --spec-id <id>
# Note the job ID from output

# 2. Stream logs to review
alethia jobs logs --id <plan-job-id> --follow

# 3. Apply
alethia apply --spec-id <id> --plan-job-id <plan-job-id>
```

### Scripting with --wait

```bash
# Block until plan completes, then apply
alethia plan --spec-id <id> --wait
alethia apply --spec-id <id> --plan-job-id <id> --wait
```

### Log Tailing

```bash
# Stream logs like tail -f
alethia jobs logs --id <job-id> --follow
```
