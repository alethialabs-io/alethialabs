# Command Reference

alethia organizes its commands into groups (`connector`, `project`, `runner`, `jobs`, `cluster`)
plus top-level auth commands. All commands communicate with the Alethia API — they do not execute
Terraform directly.

## Command Tree

```
alethia
├── login [--force]
├── logout
├── version
├── connector
│   ├── gcp [--project <id>] [--manual]
│   ├── aws [--region <r>] [--role-name <name>] [--manual]
│   ├── azure [--subscription <id>] [--manual]
│   ├── list
│   └── remove [provider] [--yes]
├── project
│   ├── list
│   ├── get [project_name]
│   ├── plan [--project-id <id>] [--runner-id <id>] [--wait]
│   ├── apply [--project-id <id>] [--runner-id <id>] [--plan-job-id <id>] [--wait]
│   └── destroy [--project-id <id>] [--runner-id <id>] [--wait]
├── runner
│   ├── deploy [--cloud-identity-id <id>] [--name <name>] [--region <r>] [--assigned-runner-id <id>] [--wait]
│   ├── list
│   ├── destroy [--runner-id <id>] [--assigned-runner-id <id>] [--wait]
│   └── remove [runner_id]
├── jobs
│   ├── list
│   ├── get <job_id>
│   ├── logs <job_id> [--follow]
│   └── cancel <job_id>
└── cluster
    └── list
```

## Interactive Selection

When `--project-id` is omitted from `project plan`, `project apply`, or `project destroy`, alethia launches an
interactive TUI selector. See [TUI Components](/cli/tui) for how this works.

## Common Patterns

### Connect a Cloud Account

```bash
# Connect a GCP project (runs the installer in your Cloud Shell)
alethia connector gcp --project my-gcp-project

# List connected accounts
alethia connector list
```

See [Connector Commands](/cli/commands/connector) for AWS and Azure flows.

### Plan-Review-Apply

```bash
# 1. Plan
alethia project plan --project-id <id>
# Note the job ID from output

# 2. Stream logs to review
alethia jobs logs <plan-job-id> --follow

# 3. Apply
alethia project apply --project-id <id> --plan-job-id <plan-job-id>
```

### Scripting with --wait

```bash
# Block until plan completes, then apply
alethia project plan --project-id <id> --wait
alethia project apply --project-id <id> --plan-job-id <id> --wait
```

### Log Tailing

```bash
# Stream logs like tail -f
alethia jobs logs <job-id> --follow
```
