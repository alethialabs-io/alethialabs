# Zone Commands

Zones are workspaces that group related infrastructure configurations. See [Domain Model](/concepts/domain-model) for how zones fit into the entity hierarchy.

## `alethia zone create`

Create a new zone with an interactive form.

```bash
alethia zone create --name <name>
```

| Flag | Required | Description |
|------|----------|-------------|
| `--name` | Yes | Zone name (unique per user) |

The command uses a [Charmbracelet `huh` form](/cli/tui) for interactive input if `--name` is omitted. On success, prints the zone ID.

## `alethia zone list`

List all zones owned by the authenticated user.

```bash
alethia zone list
```

Output:

```
ID                                   NAME          SPECS  CREATED
a1b2c3d4-...                         api-backend   2      2026-06-01
e5f6g7h8-...                         data-platform 1      2026-05-28
```

## `alethia zone delete`

Delete a zone and all its Specs.

```bash
alethia zone delete --id <zone-id>
```

| Flag | Required | Description |
|------|----------|-------------|
| `--id` | Yes | Zone UUID |

<Callout type="warn">
This deletes the zone record and all child Specs from the database. It does NOT destroy provisioned cloud resources — run `alethia destroy spec` first to tear down infrastructure.
</Callout>
