# Vineyard Commands

Vineyards are workspaces that group related infrastructure configurations. See [Domain Model](/concepts/domain-model) for how vineyards fit into the entity hierarchy.

## `alethia vineyard create`

Create a new vineyard with an interactive form.

```bash
alethia vineyard create --name <name>
```

| Flag | Required | Description |
|------|----------|-------------|
| `--name` | Yes | Vineyard name (unique per user) |

The command uses a [Charmbracelet `huh` form](/cli/tui) for interactive input if `--name` is omitted. On success, prints the vineyard ID.

## `alethia vineyard list`

List all vineyards owned by the authenticated user.

```bash
alethia vineyard list
```

Output:

```
ID                                   NAME          VINES  CREATED
a1b2c3d4-...                         api-backend   2      2026-06-01
e5f6g7h8-...                         data-platform 1      2026-05-28
```

## `alethia vineyard delete`

Delete a vineyard and all its vines.

```bash
alethia vineyard delete --id <vineyard-id>
```

| Flag | Required | Description |
|------|----------|-------------|
| `--id` | Yes | Vineyard UUID |

<Callout type="warn">
This deletes the vineyard record and all child vines from the database. It does NOT destroy provisioned cloud resources — run `alethia destroy vine` first to tear down infrastructure.
</Callout>
