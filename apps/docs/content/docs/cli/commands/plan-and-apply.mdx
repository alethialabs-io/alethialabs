# Plan & Apply

These `project` subcommands queue infrastructure operations that [Runners](/runner) execute. They
implement the Plan-Review-Apply workflow from the terminal.

## `alethia project plan`

Queue an [OpenTofu](/concepts/glossary) plan job — the Runner runs `tofu plan`.

```bash
alethia project plan [--project-id <id>] [--runner-id <id>] [--wait]
```

| Flag | Required | Description |
|------|----------|-------------|
| `--project-id` | No | Project UUID (interactive selector if omitted) |
| `--runner-id` | No | Specific Runner to execute (any available if omitted) |
| `--wait` | No | Block until the job completes |

On success, prints the job ID. Use `alethia jobs logs <id> --follow` to stream the plan output.

## `alethia project apply`

Queue a deployment (apply) job.

```bash
alethia project apply [--project-id <id>] [--runner-id <id>] [--plan-job-id <id>] [--wait]
```

| Flag | Required | Description |
|------|----------|-------------|
| `--project-id` | No | Project UUID (interactive selector if omitted) |
| `--runner-id` | No | Specific Runner to execute |
| `--plan-job-id` | No | Reference a specific plan to apply |
| `--wait` | No | Block until the job completes |

The `--plan-job-id` flag ensures the apply uses the exact plan you reviewed. If the Project configuration changed since the plan was generated, the job fails with a [config hash mismatch](/console/jobs/plan-and-apply).

<Callout type="info">
A DEPLOY job is `tofu plan` → **[verify](/concepts/glossary) gate** → `tofu apply` → ArgoCD scaffold.
The verify gate is **fail-closed**: a hard control failure blocks *before* `tofu apply` unless an
authorized override waives it. The report is attached to the job's `execution_metadata` and rendered
in the console Plan tab — inspect it with `alethia jobs logs <apply-job-id> --follow`.
</Callout>

## `alethia project destroy`

Destroy a Project's provisioned infrastructure.

```bash
alethia project destroy [--project-id <id>] [--runner-id <id>] [--wait]
```

Queues a DESTROY job. The Runner runs `tofu destroy` to tear down all resources. OpenTofu state is preserved (it lives behind the Console state proxy, not on the Runner), so accidental destroys can be recovered by re-applying.

<Callout type="info">
To tear down a deployed **Runner** (not a Project), use [`alethia runner destroy`](/cli/commands/runners).
</Callout>

## Interactive Selection

When `--project-id` is omitted, alethia launches a TUI selector:

1. Select a Project from the list
2. The command proceeds with the selected Project

See [TUI Components](/cli/tui) for details on the interactive selectors.

## Scripting Examples

```bash
# Plan and wait, then apply
JOB_ID=$(alethia project plan --project-id $PROJECT_ID --wait 2>&1 | grep "Job ID" | awk '{print $3}')
alethia project apply --project-id $PROJECT_ID --plan-job-id $JOB_ID --wait

# Destroy with wait
alethia project destroy --project-id $PROJECT_ID --wait
```
