# IaC Commands

The `iac` group shows the **BYO Terraform/OpenTofu source** attached to an environment — your own
IaC module, pulled from a connected git repo, that Alethia applies for that environment.

<Callout type="info">
The CLI is a client of the same Console API. `iac show` calls `/cli/projects/<project>/byo-iac`
with your [CLI token](/cli/authentication) and operates on the **active organization**.
</Callout>

## `alethia iac show`

Show the BYO IaC source attached to a project environment.

```bash
alethia iac show --project my-project
alethia iac show -p my-project -e staging
alethia iac show -p my-project -o json
```

| Flag | Required | Description |
|------|----------|-------------|
| `--project`, `-p` | Yes | Project name or id. |
| `--env`, `-e` | No | Environment name, stage, or id. Omit to use the project's default environment. |

Renders a field/value card: **name**, **repo**, **path**, **ref**, **enabled**, **scan** (the
IaC-safety scan lifecycle), **pinned commit** (what a deploy will apply), **deployed commit** (what
the last deploy applied), **status**. Prints `No BYO IaC source attached.` when none is configured.
Add `-o json` for the full source object (the elench scan report itself is console-only). Backing
call: `GET /cli/projects/<project>/byo-iac`.
