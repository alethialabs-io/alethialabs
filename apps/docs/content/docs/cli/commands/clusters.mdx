# Cluster Commands

The command is `alethia cluster` (singular) with a `clusters` alias, so both forms work
interchangeably.

## `alethia cluster list`

List all provisioned Kubernetes clusters.

```bash
alethia cluster list
# or, equivalently:
alethia clusters list
```

Output:

```
PROJECT               CLUSTER                 VERSION  STATUS     ARGOCD    NODES  REGION      COST
api-backend (prod)    api-backend-production  1.30     ● active   exposed   1/3/5  eu-west-1   $210/mo
data-pipeline (stg)   data-pipeline-staging   1.29     ● active   port-fwd  1/2/3  us-east4    $95/mo
```

Shows clusters across your projects with:
- Project (with its environment) and cluster name
- Kubernetes version
- Status — the status message is shown inline when a cluster is degraded or failed
- ArgoCD access — `exposed` (a managed-ingress URL exists) or `port-fwd`
- Node count (min/desired/max)
- Region
- Estimated monthly cost

Pass `-o json` for the full records (including each cluster's ArgoCD URL, instance types, and node sizes).

## Accessing a Cluster

After identifying a cluster, configure kubectl using the provider-specific command:

<Tabs items={["AWS", "GCP", "Azure"]}>
  <Tab value="AWS">
    ```bash
    aws eks update-kubeconfig --name api-backend-production --region eu-west-1
    kubectl get nodes
    ```
  </Tab>
  <Tab value="GCP">
    ```bash
    gcloud container clusters get-credentials data-pipeline-staging --region us-east4
    kubectl get nodes
    ```
  </Tab>
  <Tab value="Azure">
    ```bash
    az aks get-credentials --name my-cluster --resource-group my-rg
    kubectl get nodes
    ```
  </Tab>
</Tabs>
