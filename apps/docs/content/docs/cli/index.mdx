# alethia CLI

alethia is the command-line interface for the platform. It provides terminal-based access to all infrastructure operations — planning and deploying Projects, monitoring jobs, and managing Runners.

## Overview

| Property | Value |
|----------|-------|
| Language | Go |
| Framework | [Cobra](https://github.com/spf13/cobra) (commands) + [Charmbracelet](https://charm.sh/) (TUI) |
| Version | Managed by release-please (`alethia-v*` tags) |
| Distribution | Homebrew · Scoop · install script (get.alethialabs.io) · .deb/.rpm/.apk · Docker · GitHub Releases (macOS/Linux/Windows, amd64/arm64) |

## How It Works

alethia is a **thin client** that communicates with the Alethia API over HTTPS. It does not run Terraform directly — instead, it queues jobs that [Runners](/runner) execute. The CLI handles:

1. **Authentication** — [Device code flow](/concepts/authentication) with tokens stored locally
2. **Configuration** — Creating and inspecting Projects
3. **Job submission** — Queuing Plan / Deploy / Destroy jobs
4. **Monitoring** — Streaming logs, checking job status, waiting for completion
5. **Management** — Listing clusters, managing Runners

## Global Flags

Every command accepts two persistent flags:

| Flag | Default | Description |
|------|---------|-------------|
| `-o, --output <table\|json\|csv>` | `table` | Output format. `json`/`csv` print static, machine-readable output. |
| `--no-input` | off | Disable interactive prompts; fail instead of prompting. Auto-enabled when stdin is not a TTY. |

List/get commands render an **interactive table** only on a TTY with the default `table` format.
With `json`, `csv`, a pipe, or `--no-input`, they print static output instead — ideal for scripting:

```bash
alethia project list -o json | jq '.[].name'
```

See [Identity & Global Options](/cli/identity) for details.

## Quick Start

```bash
# Install
brew install alethialabs-io/tap/alethia

# Point at your control plane and authenticate
export ALETHIA_WEB_ORIGIN=https://app.alethialabs.io
alethia login

# Confirm who you are and pick an active org
alethia whoami
alethia org switch

# Connect a cloud account (AWS, GCP, or Azure)
alethia connector gcp --project my-gcp-project

# Plan infrastructure (Projects are created in the console)
alethia project plan --project-id <id>

# Stream logs
alethia jobs logs <job-id> --follow

# Deploy
alethia project apply --project-id <id> --plan-job-id <plan-job-id>
```
