# TUI Components

alethia uses the [Charmbracelet](https://charm.sh/) suite for rich terminal UI. Three libraries are used:

| Library | Purpose |
|---------|---------|
| [**huh**](https://github.com/charmbracelet/huh) | Interactive forms and selection prompts |
| [**bubbletea**](https://github.com/charmbracelet/bubbletea) | Full TUI framework (used for the login spinner) |
| [**lipgloss**](https://github.com/charmbracelet/lipgloss) | Terminal styling and layout |

## Interactive Selectors

The interactive selectors are **not a standalone command** — they appear automatically whenever a
command needs a resource you didn't specify on the command line and prompting is allowed (a TTY with
`--no-input` off). For example, omitting `--project-id` from `project plan`, `project apply`, or
`project destroy` opens a Project picker; omitting `--runner-id` opens a Runner picker; and
`org switch` / `connector remove` with no argument open their own pickers.

### Project Selector

```
? Select a Project:
  > production / eu-west-1 (ACTIVE)
    staging / eu-west-1 (DRAFT)
    development / us-east-1 (PROVISIONING)
```

Each Project shows its environment stage, region, and current status.

<Callout type="info">
With `--no-input` (or when stdin is not a TTY — e.g. in CI or a pipe), the selectors are disabled.
Pass the required IDs explicitly (`--project-id`, `--runner-id`, an org argument, …) or the command
fails with a clear "interactive input required" error instead of hanging.
</Callout>

## Progress Stepper

The visual progress stepper shows deployment phase progress using diamond icons:

```
◆ Authentication     ← completed
◆ Configuration      ← completed
◈ OpenTofu Plan      ← current (running)
◇ Cost Analysis      ← pending
◇ Reporting          ← pending
```

- `◆` (filled diamond) — completed step
- `◈` (diamond with dot) — currently executing
- `◇` (empty diamond) — pending

## Login Spinner

The `alethia login` command uses a Bubble Tea model for the authentication flow:

```
🔐 Waiting for browser authentication...

  Visit: https://alethia.example.com/cli/login?device_code=abc123
  Enter the verification code shown in your browser:

  > _

  ⠋ Waiting for approval...
```

The spinner animates while polling the exchange endpoint every 2 seconds.

## Config Printer

Job details and Project configurations are displayed with formatted JSON output using lipgloss styling:

- Monospace font for values
- Color-coded keys
- Indented structure for nested objects
- Truncated long values with ellipsis
