Сигурност

Как работи без ключове?

Платформата не съхранява облачни credentials. Tendril поема временна роля при всяко изпълнение.

AWS

Cross-Account IAM Role

Потребителят деплойва CloudFormation шаблон. Създава се IAM роля с External ID. Tendril поема ролята чрез STS AssumeRole. Сесия: 1 час.

GCP

Workload Identity Federation

OIDC токен от Trellis се обменя за временни GCP credentials. Без service account JSON ключове.

Azure

Federated Identity

App Registration с федерирани credentials. OIDC автентикация. Без client secrets.

CLI автентикация: RFC 8628 Device Authorization Flow. Потребителят потвърждава в браузъра — CLI получава Refresh Token, криптиран локално. Без пароли в терминала.
